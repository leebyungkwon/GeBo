# UX Heuristics Checklist (Nielsen Norman Group)

## 1. Visibility of System Status
- [ ] **Feedback**: 사용자의 액션(클릭, 저장 등)에 대해 0.1초 이내에 즉각적인 피드백(스피너, 토스트)을 제공하는가?
- [ ] **Progress**: 긴 작업(>1초) 진행 시 프로그래스 바나 단계 표시를 제공하는가?

## 2. Match Between System and Real World
- [ ] **Language**: 'Database Error' 같은 시스템 용어 대신 '저장에 실패했습니다' 같은 사용자 언어를 사용하는가?
- [ ] **Metaphor**: 아이콘(휴지통, 돋보기)이 현실 세계의 관습과 일치하는가?

## 3. User Control and Freedom
- [ ] **Undo/Redo**: 사용자가 실수했을 때 되돌리기(Undo)나 취소(Cancel)를 쉽게 할 수 있는가?
- [ ] **Exit**: 팝업이나 모달에 명확한 닫기(X) 버튼이나 '바깥 클릭해서 닫기'가 있는가?

## 4. Consistency and Standards
- [ ] **Internal**: 같은 기능을 하는 버튼은 위치와 색상이 동일한가?
- [ ] **External**: 플랫폼 관습(예: 모바일 하단 탭, 데스크탑 상단 GNB)을 따르는가?

## 5. Error Prevention
- [ ] **Confirmation**: 삭제(Delete) 같은 파괴적 액션 전에 확인 팝업을 띄우는가?
- [ ] **Constraints**: 날짜 입력 시 달력을 제공하여 포맷 오류를 원천 차단하는가?

## 6. Recognition Rather Than Recall
- [ ] **Context**: 사용자가 기억하지 않아도 되도록, 필요한 정보를 화면에 항상 표시하는가? (예: 검색 결과 상단에 검색어 표시)
- [ ] **Menu**: 메뉴 항목이 숨겨져 있지 않고 찾기 쉬운가?

## 7. Flexibility and Efficiency of Use
- [ ] **Shortcuts**: 숙련된 사용자를 위한 단축키나 제스처를 지원하는가?
- [ ] **Personalization**: 자주 쓰는 기능을 대시보드에 배치할 수 있는가?

## 8. Aesthetic and Minimalist Design
- [ ] **Signal-to-Noise**: 불필요한 장식이나 정보가 주된 콘텐츠를 방해하지 않는가?
- [ ] **Visual Hierarchy**: 중요한 정보가 더 크고 진하게 표시되는가?

## 9. Help Users Recognize, Diagnose, and Recover from Errors
- [ ] **Plain Language**: 에러 메시지가 '무엇이 문제인지'와 '어떻게 해결하는지'를 알려주는가?
- [ ] **Visual Cue**: 에러 필드를 붉은색 테두리 등으로 명확하게 강조하는가?

## 10. Help and Documentation
- [ ] **Onboarding**: 복잡한 기능에 대해 첫 사용 가이드나 툴팁을 제공하는가?
- [ ] **Searchable**: 도움말 문서를 쉽게 검색할 수 있는가?
