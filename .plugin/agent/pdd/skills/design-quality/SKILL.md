# Design Quality Skill Instructions

## 1. When to use this skill
- Before finalizing `ui_guide.md` or `design_options.md`.
- To ensure the design is usable (UX), systematic (Tokens), and alive (Motion).

## 2. Resources
### ✅ Checklists
- **Accessibility (A11y)**: `checklists/accessibility.md` (WCAG 2.1 AA)
- **UX Heuristics**: `checklists/ux_heuristics.md` (10 Usability Principles)

### 📄 Templates
- **Design Tokens**: `templates/design_token_spec.md` (Atomic Naming)
- **Responsive Guide**: `templates/responsive_guide.md` (Breakpoints)

### 📘 Guides
- **Motion & Interaction**: `guides/motion_interaction.md` (Animation Rules)

## 3. How to Apply
1.  **Draft Design**: Create your initial design option.
2.  **Verify**: Run A11y and UX Heuristics checklists.
3.  **Systemize**: Define colors and fonts using the Design Token Template.
4.  **Polish**: Add interaction details using the Motion Guide.
