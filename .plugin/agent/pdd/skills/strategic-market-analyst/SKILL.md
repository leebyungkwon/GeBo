# Strategic Market Analyst Skill Instructions

## 1. 사용 시점
- 신규 기능 기획 혹은 화면 개편 전 (Step 01. Phase 1~2).
- 사용자 요청 사항을 넘어선 '혁신 제안'이 필요할 때.

## 2. 전략적 리서치 프로세스
1. **페인 포인트 사냥 (Pain-point Hunting)**:
   - `"[Competitor Name] complaints"`, `"[Competitor Name] 리뷰 불만"`, `"site:reddit.com [Service Category] problem"` 등의 쿼리로 실제 사용자의 고충을 찾습니다.
2. **혁신 포인트로 치환 (Inversion)**:
   - 찾은 불만 사항을 우리 제품에서 어떻게 해결할지 정의합니다.
   - 예: "로그인이 너무 느림" -> "Biometric / Social 간편 인증 최우선 도입 및 1초 이내 진입 설계".
3. **비즈니스 가치 정량화**:
   - 도입 시 예상되는 운영 효율성이나 사용자 만족도 지표(Success Metric)를 함께 도출합니다.

## 3. 결과물 반영
- 도출된 인사이트를 `plan_{page}.md`의 **"4. [Proactive Update] 혁신 및 개선 제안"** 섹션에 정교하게 담아냅니다.
