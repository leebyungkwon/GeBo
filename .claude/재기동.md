# 프로젝트 재기동 가이드

코드 수정 사항(특히 백엔드)을 반영하기 위해 FE와 BE를 모두 재기동하는 방법입니다.

루트 디렉토리에 생성된 배치 파일을 실행하면 기존 프로세스를 종료하고 자동으로 다시 시작합니다.

- **모두 재기동**: `.\restart.bat` (FE+BE)
- **백엔드만**: `.\restart_be.bat` (BE)
- **프론트엔드만**: `.\restart_fe.bat` (FE)

```powershell
cd c:\Users\Administrator\.gemini\workspace\Bo
# 필요한 스크립트 실행
.\restart_be.bat
```

## 1. 수동 프로세스 종료 (PowerShell)

터미널에서 아래 명령어를 실행하여 3002(FE)와 8002(BE) 포트를 사용하는 프로세스를 강제 종료합니다.

```powershell
# 3002 포트 (FE) 종료
Stop-Process -Id (Get-NetTCPConnection -LocalPort 3002).OwningProcess -Force -ErrorAction SilentlyContinue

# 8002 포트 (BE) 종료
Stop-Process -Id (Get-NetTCPConnection -LocalPort 8002).OwningProcess -Force -ErrorAction SilentlyContinue
```

## 2. 서버 실행

### 백엔드 (API Server)
```powershell
cd c:\Users\Administrator\.gemini\workspace\Bo\bo-api
./gradlew.bat bootRun --args='--spring.profiles.active=local'
```

### 프론트엔드 (Admin FE)
```powershell
cd c:\Users\Administrator\.gemini\workspace\Bo\bo
npm run dev
```

---
> [!TIP]
> 백엔드 수정 사항이 반영되지 않을 경우 `./gradlew.bat clean classes`를 먼저 실행한 후 `bootRun`을 시도해 보세요.
