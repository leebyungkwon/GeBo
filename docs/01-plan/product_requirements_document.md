# 백오피스(Admin) 제품 요구사항 정의서 (PRD)

## 1. 개요 및 배경 (Overview & Background)
- **Problem Statement (해결하고자 하는 문제)**: 북미 시장(`geamerica.com`)을 시작으로 글로벌 Region마다 콘텐츠와 이벤트를 체계적으로 관리해야 하지만, 체계적인 CMS(콘텐츠 관리 시스템)가 부재하여 프론트엔드 배포 작업이 동반되는 등 관리자의 운영 리소스가 낭비되고 있다.
- **Business Goal (비즈니스 목표/기대 효과)**: 
  - 코딩 지식이 없는 기획자/마케터가 개발자의 도움 없이 팝업, 배너, 메뉴, 일반 콘텐츠를 즉시 등록 및 수정할 수 있게 한다.
  - 검색엔진최적화(SEO)를 위한 `[slug]` 및 `[meta info]` 데이터 제어권을 운영 관리자에게 완전히 부여한다.
  - KPI(방문자 로그, 트래픽 등)를 한눈에 볼 수 있는 통계 대시보드를 제공한다.
  - 향후 다국어(글로벌 지사) 확장 시 기존 북미 사이트의 DB 콘텐츠를 원활하게 확산(Clone)할 수 있는 구조를 마련한다.

## 2. 타겟 유저 페르소나 (Target Audience)
- **Primary User (운영 관리자 / 마케터)**: 
  - **특징**: 웹 코드(HTML/CSS)에 대한 지식이 없는 비개발 직군. 새로운 캠페인 배너나 제품 공지사항을 사이트에 빠르게 노출시켜야 함.
  - **Needs**: 복잡한 승인 절차(결재) 없이 저장 시 즉시 프론트엔드 화면에 반영(Direct Publish)되기를 원함.
- **Secondary User (최고 관리자 / 시스템 어드민)**: 
  - **특징**: 전체 시스템과 권한을 통제하는 마스터 계정.
  - **Needs**: 하급 관리자(운영진)의 계정을 생성/삭제하고, 통합 트래픽 통계를 열람하길 원함.

## 3. 핵심 유저 스토리 (User Stories)
- **[운영 관리자]**로서, 나는 **[게시글 및 페이지의 slug와 메타 태그(Title, Description)를 직접 수정]**하고 싶다. 왜냐하면 **[검색 엔진 노출(SEO) 성과를 즉각적으로 개선하기]** 위해서이다.
- **[최고 관리자]**로서, 나는 **[메인 대시보드에서 방문자 통계와 KPI 차트를 확인]**하고 싶다. 왜냐하면 **[글로벌 마케팅 성과를 한눈에 트래킹하기]** 위해서이다.
- **[시스템 어드민]**로서, 나는 **[향후 생성될 타 국가(Region) 카테고리로 기존 콘텐츠를 복사/배포]**하고 싶다. 왜냐하면 **[글로벌 지사 확장 시 투입되는 콘텐츠 세팅 시간을 절약하기]** 위해서이다.

## 4. 백오피스 메뉴 구조도 (Information Architecture - Menu List)
시스템 접속 시 좌측 영역(LNB)에 고정으로 노출될 뎁스(Depth)별 메뉴 목록입니다.

*   🏠 **대시보드**
    *   **통계 요약**: 일별 방문자, 많이 본 페이지, 최근 생성된 콘텐츠 현황 수치
*   📚 **기본 콘텐츠 관리**
    *   **팝업 관리**: 노출 날짜 및 이미지/링크 매핑
    *   **배너 관리**: 메인 슬라이더 등 배너 리스트 등록 및 노출 순서 변경
*   💬 **게시판(Board) 관리**
    *   **게시판 마스터 구성**: 게시판 카테고리(예: 공지사항, PR News, FAQ) 생성
    *   **게시글 관리**: 생성된 카테고리별 게시글 목록 및 상세 폼 에디터
*   ⚙️ **사이트 메뉴/SEO 셋업**
    *   **메뉴 및 라우터 관리**: 실제 프론트엔드 GNB/LNB의 트리 구조 뎁스 세팅 및 링크 연결
    *   **고정 페이지 메타정보 관리**: About Us 등 고정된 페이지의 SEO(Title, Desc) 일괄 수정
*   🛡️ **관리자 설정 (Super Admin 전용)**
    *   **계정 및 권한 관리**: 서브 계정 생성 및 조회 전용/편집 가능 권한 부여

## 5. 기능 범위 상세 명세 (Feature Scope - MoSCoW)

### 🔴 Must Have (MVP 필수 핵심 기능 - 상세 데이터 셋)

#### 1. 팝업 및 배너 관리 (Popup & Banner CMS)
- **데이터 필드**: 제목, 이미지 파일 첨부(PC/Mobile 개별 업로드 가능), 클릭 시 이동할 링크 URL, 노출 시작일시, 노출 종료일시, 활성화(On/Off) 토글.
- **인수 조건(Acceptance Criteria)**:
  - 노출 종료 일시가 지났거나 비활성 상태인 항목은 사용자 프론트엔드에서 즉시 사라져야 한다.
  - 리스트 위젯에서 드래그 앤 드롭(Drag & Drop)이나 상하 화살표를 통해 롤링 순서를 정렬할 수 있어야 한다.

#### 2. 메뉴 및 라우팅 관리 (Menu & Routing Control)
- **데이터 필드**: 메뉴명, 상위 메뉴 지정(Parent ID 지정하여 Depth 구성), 링크 URL 혹은 게시판 모듈 ID.
- **인수 조건**:
  - GNB(상단 네비게이션) 구조를 백오피스에서 추가/수정/삭제 시 프론트엔드의 네비게이션 바가 레이아웃 깨짐 없이 갱신되어야 한다.

#### 3. 콘텐츠 및 SEO 관리 (Content & Meta Info)
- **데이터 필드**: 제목, 작성일, 본문(Rich Text HTML), **Custom Slug**, **Meta Title**, **Meta Description**, **Open Graph Image**.
- **인수 조건**:
  - TinyMCE 등 위지윅(WYSIWYG) 에디터를 제공하여 본문 내 이미지 삽입과 텍스트 스타일링이 가능해야 한다.
  - 작성자가 `/news/our-latest-product` 형식으로 **Slug**를 직접 타이핑하여 입력할 수 있어야 한다. (미입력 시 제목 기반 자동 치환)

#### 4. 게시판 관리 (Board Management)
- **데이터 필드**: 게시판 이름, 첨부파일 허용 여부, 스킨 타입(목록형/썸네일형).
- **인수 조건**:
  - 마스터 권한자가 새로운 'PR 센터' 라벨을 가진 게시판을 생성하면, 좌측 메뉴에 즉시 해당 게시판 관리 링크가 동적으로 생성되어야 한다.

#### 5. 관리자 권한 관리 (RBAC)
- **데이터 필드**: ID(이메일 계정), 이름, 비밀번호, 권한 등급 콤보박스(`SUPER_ADMIN`, `EDITOR`).
- **인수 조건**:
  - `EDITOR` 레벨의 계정으로 로그인한 경우, 좌측 메뉴에서 '관리자 설정' 및 '게시판 마스터 구성' 메뉴 자체가 보이지 않아 접근을 원천 차단해야 한다.

### 🟡 Should Have (권장/부가 기능)

#### 1. 자체 통계 대시보드 (KPI Dashboard)
- 외부 GA4 창을 띄우지 않고, 백오피스 메인 화면에 자체 수집 또는 GA4 API를 연동한 일별 트래픽 그래프와 페이지 뷰(PV) Top 5 순위 표를 렌더링한다. (단, 1단계 MVP 특성상 복잡도에 따라 GA4 링크 제공으로 대체 가능)

#### 2. 프론트엔드 내 직접 편집 (In-context Edit)
- 관리자가 로그인한 브라우저 세션 상태에서 실제 `geamerica.com` 프론트 화면에 접속하면 텍스트 박스 근처에 [수정 연필 아이콘]이 나타나며, 오버레이 모달 창에서 내용 편집 후 즉시 DB에 적용된다.

### 🟢 Could Have (차기 마일스톤)
#### 다중 국가 콘텐츠 자동 확산 (Content Localization Sync)
- 북미 사이트 DB에 등록된 데이터를 선택하여 '남미 사이트 DB' 공간으로 클릭 한 번에 복사(Clone) 인서트하는 글로벌 프로비저닝 기능.

## 6. 성공 지표 (Success Metrics)
- **정량적 (Quantitative)**: 마케터의 콘텐츠 추가 및 배너 교체 시 필요했던 개발팀 JIRA 요청 티켓 건수 **80% 이상 감소**.
- **정성적 (Qualitative)**: 신규 이벤트 배너를 사이트에 퍼블리싱하는 End-to-End 물리적 소요 시간이 **개발자 배포 주기 포함 수일(Days) 단위에서 즉각(Minutes) 등록으로 지연 없이 처리**됨.
