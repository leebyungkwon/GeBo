# [페이지 기획] 관리자 계정 관리 (Admin User Management) PRD

## 1. 개요 및 배경 (Overview & Background)
- **목적**: 시스템 접근 권한을 가진 관리자 계정을 효율적으로 생성, 수정, 삭제하고 적절한 권한(RBAC)을 부여하기 위함.
- **해결 과제**: 현재 DB 직접 조작을 통해서만 계정 관리가 가능하므로, 이를 운영 대시보드 내에서 안전하고 편리하게 처리할 수 있는 UI/UX가 필요함.

## 2. 타겟 유저 페르소나 (Target Audience)
- **Primary User (최고 관리자)**: 시스템 전체를 통제하며 운영진 계정을 관리함.
- **Secondary User (운영 관리자)**: 본인의 계정 정보 확인 및 권한에 따른 기능 수행.

## 3. 유저 스토리 (User Stories)
- [최고 관리자]로서, 나는 [사번(Employee ID)을 포함한 관리자 계정을 생성]하고 싶다. 왜냐하면 [사내 인사 정보와 연동하여 관리 효율성을 높이기] 위해서이다.
- [최고 관리자]로서, 나는 [비밀번호를 직접 입력하거나 시스템에서 임시 비밀번호를 발급]하고 싶다. 왜냐하면 [보안 정책에 따라 유연하게 대응하기] 위해서이다.
- [최고 관리자]로서, 나는 [기존 관리자의 권한 등급을 변경]하고 싶다. 왜냐하면 [담당 업무 변경에 따른 접근 제어가 필요하기] 때문이다.
- [최고 관리자]로서, 나는 [퇴사한 관리자의 계정을 삭제/비활성화]하고 싶다. 왜냐하면 [시스템 보안을 유지하기] 위해서이다.

## 4. 기능 범위 (Feature Scope - MoSCoW)
- **Must Have (필수)**:
    - 관리자 목록 조회 (이름, 이메일, **사번**, 권한, 생성일, 최종 로그인 시간).
    - 신규 관리자 등록 (**사번 입력 필드**, 이메일 중복 체크 포함).
    - **비밀번호 설정 옵션**: 직접 입력 기능 + 임시 비밀번호 자동 생성 및 발급 기능.
    - 관리자 정보 수정 (이름, **사번**, 권한).
    - 관리자 계정 삭제.
- **Should Have (필수)**:
    - 관리자 검색 기능 (이름/이메일 기준).
    - 권한별 필터링 기능.
- **Could Have (필수)**:
    - 관리자 접속 로그 조회.
    - 계정 잠금/해제 기능.
- **Won't Have (제외)**:
    - 일반 사용자(Customer) 관리 (본 페이지의 범위를 벗어남).

## 5. 비즈니스 정책 (Business Logic)
- **사번(Employee ID)**: 필수 입력 항목이며, 시스템 내에서 고유해야 함.
- **비밀번호 정책**: 
    - 직접 입력 시: 최소 8자 이상, 영문/숫자/특수문자 조합 권장.
    - 임시 비밀번호 발급 시: 임의의 12자 문자열 생성 후 화면에 노출 (추후 이메일 발송 기능 확장 고려).
