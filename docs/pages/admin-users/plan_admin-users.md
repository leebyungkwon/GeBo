# [사용자 관리] 단위 화면 상세 기획서 (Detailed Functional Spec)

## 1. 개요 및 비즈니스 맥락 (Background & Goals)
- **Problem Statement (해결 과제)**: 백오피스 운영을 위해 관리자 계정을 효율적으로 생성, 수정, 삭제하고 적절한 권한을 부여할 수 있는 통합 관리 체계가 필요합니다. 기존 화면은 기능 중심적으로 구성되어 있어, 운영 편의성과 보안 정책(임시 비번 등)이 강화된 새로운 설계가 요구됩니다.
- **Business Objective (목표)**: 보안이 강화된 관리자 계정 생태계를 구축하고, 운영자의 계정 관리 실수를 최소화하며, 권한 오남용을 방지합니다.
- **Success Metric (성공 지표)**: 신규 관리자 등록 및 권한 설정 시간 30% 단축, 계정 보안 사고 0건.

## 2. 정보 구조 및 메뉴 설계 (IA & Menu Planning)
- **모듈 내 전체 화면 목록**:
    - 관리자 계정 목록 (Main List)
    - 관리자 상세 조회/수정 (Drawer/Modal)
    - 신규 관리자 등록 (Drawer/Modal)
    - 계정 활동 이력 (History - 추후 확장)
- **메뉴 경로 (Menu Path)**: 사이트 설정 -> 관리자 설정 -> 사용자 관리
- **사용자 권한 정책**: 
    - **SUPER_ADMIN**: 모든 관리자 계정 조회, 등록, 수정, 삭제 권한 보유.
    - **EDITOR**: 본인 계정 정보 확인 및 제한된 목록 조회 가능 (수정/삭제 불가).

## 3. 화면 상세 명세 및 핵심 기능 (Detailed Screen Specs)
- **Layout Structure**: 
    - 상단: 타이틀, 통계 요약(전체/활성/잠금), 신규 등록 버튼
    - 중단: 검색 및 필터바 (이름/이메일/사번 검색, 권한별 필터)
    - 하단: 관리자 목록 그리드 (Table), 페이징
    - 우측: 등록/수정용 슬라이드 오버 (Drawer)

- **Indispensable Feature List (필수 기능)**:
    - **스마트 검색**: 검색어 입력 시 즉시 필터링 (Debounce 적용).
    - **임시 비밀번호 자동 발급**: 신규 등록 시 보안을 위해 임시 비밀번호를 생성하여 알림 제공.
    - **계정 상태 관리**: 활성/비활성 상태 전환 기능.
    - **권한 매핑**: SUPER_ADMIN, EDITOR 등 역할 지정.

- **Input Fields & Form Control**: 
    - 이름 / String / 필수 / -
    - 이메일 (ID 겸용) / Email / 필수 / - (중복 체크 필수)
    - 사번 / String / 필수 / - (중복 체크 필수)
    - 권한 / Enum / 필수 / EDITOR (기본값)
    - 계정 상태 / Boolean / 필수 / 활성 (기본값)

## 4. [Proactive Update] 혁신 및 개선 제안 (Enhancement Suggestions)
- **아이디어 1**: **"마지막 로그인 IP 및 시간 표시"** - 보안 모니터링을 위해 목록에서 마지막 접속 정보를 직관적으로 노출합니다.
- **아이디어 2**: **"일괄 상태 변경(Bulk Action)"** - 여러 계정을 선택하여 한 번에 비활성화하거나 삭제할 수 있는 기능을 추가하여 운영 효율을 높입니다.

## 5. 비즈니스 로직 및 예외 정책 (Logic & Exception Policy)
- **상세 비즈니스 정책**:
    - 이메일과 사번은 시스템 내에서 유일해야 합니다.
    - 신규 등록 시 비밀번호는 시스템에서 난수로 생성하며, 첫 로그인 시 변경을 강제합니다.
    - 슈퍼 관리자(자신)는 스스로를 삭제할 수 없습니다.
- **입력 검증 (Validation) 정책**: 
    - 이메일: 표준 이메일 형식 체크.
    - 이름: 2자 이상 20자 이하.
    - 사번: 숫자 및 영문 조합, 특수문자 불가.
- **예외 처리 (Edge Case Policy)**:
    - 데이터 없음: "등록된 관리자가 없습니다. 첫 관리자를 등록해 주세요." 메시지 노출.
    - 서버 오류: "통신 중 오류가 발생했습니다. 잠시 후 다시 시도해 주세요." 토스트 알림.

## 6. 공통 모듈 및 자산화 전략 (Commonality & Assets)
- **공통 UI 모듈**: `CommonTable`, `CommonDrawer`, `StatusBadge` 등을 프로젝트 표준 라이브러리로 자산화합니다.
- **공통 로직 모듈**: API 에러 처리 로직 및 데이터 포맷팅(날짜 등) 유틸리티 활용.

## 7. 디자인/개발 핸드오버 체크리스트 (Step 2 & 3 Support)
- **디자인 필수 표현**: 역할별 뱃지 컬러 구분 (SUPER: Blue, EDITOR: Grey), 계정 상태 토글 스위치.
- **기능적 인터랙션**: 등록 완료 후 목록 최상단에 즉시 반영 및 하이라이트 효과.
- **성공 인수 조건 (Acceptance Criteria)**:
    - 중복되지 않는 이메일/사번으로 계정 생성 성공.
    - 생성된 계정으로 로그인 가능(임시 비번 활용).
    - 필터링 및 검색 기능이 대소문자 구분 없이 정확히 동작.
