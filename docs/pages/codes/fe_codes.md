# 공통코드 관리 FE 상세 설계서 (Service Screen Plan)

## 1. 개요 및 퍼블리싱 자산 연결

- **설계 목적**: 시스템 전반에서 사용되는 코드성 데이터(상태/분류/우선순위 등)의 CRUD를 프론트엔드 레벨에서 보장
- **참조 자산**:
  - 페이지: `bo/src/app/admin/settings/codes/page.tsx`
  - 컴포넌트: `bo/src/components/codes/CodeGroupList.tsx`, `CodeDetail.tsx`, `CodeDetailTable.tsx`
  - 상태관리: `bo/src/store/useCodeStore.ts`

## 2. 데이터 마스터 및 유효성 검사 (Master Validation Table)

### 2.1 CodeGroup 엔티티

| 필드명 (UI) | Key | 타입 | 필수 | 검증 규칙 | 에러 메시지 |
|:---|:---|:---|:---|:---|:---|
| 그룹코드 | `groupCode` | String | Y | 영문 대문자+숫자+_ 1~30자 `^[A-Z0-9_]{1,30}$` | 그룹코드는 영문 대문자, 숫자, _만 사용 가능합니다. (최대 30자) |
| 그룹명 | `groupName` | String | Y | 1~50자 | 그룹명을 입력해주세요. |
| 설명 | `description` | String | N | 최대 200자 | - |
| 사용여부 | `active` | Boolean | Y | true/false | - |

### 2.2 CodeDetail 엔티티

| 필드명 (UI) | Key | 타입 | 필수 | 검증 규칙 | 에러 메시지 |
|:---|:---|:---|:---|:---|:---|
| 코드값 | `code` | String | Y | 영문 대문자+숫자+_ 1~30자 `^[A-Z0-9_]{1,30}$` | 코드값을 입력해주세요. |
| 코드명 | `name` | String | Y | 1~50자 | 코드명을 입력해주세요. |
| 정렬순서 | `sortOrder` | Number | Y | 양의 정수 1~999 | 정렬 순서는 1 이상의 정수를 입력해주세요. |
| 사용여부 | `active` | Boolean | Y | true/false | - |
| 설명 | `description` | String | N | 최대 200자 | - |

### 2.3 필드별 상세 Validation 시나리오

#### 2.3.1 그룹코드 (`groupCode`)

| 검증 시점 | 조건 | 결과 | 에러 메시지 |
|:---|:---|:---|:---|
| onChange | 빈 문자열 | 에러 표시 | 그룹코드를 입력해주세요. |
| onChange | 소문자 입력 | 자동 대문자 변환 | - |
| onChange | 30자 초과 | 입력 차단 (maxLength) | - |
| onChange | 특수문자(-, 공백, 한글 등) | 에러 표시 | 영문 대문자, 숫자, _만 사용 가능합니다. |
| 저장 시 | 동일 groupCode 존재 | API 409 → 에러 토스트 | 이미 동일한 그룹코드가 존재합니다. |
| 생성 후 | 수정 시도 | 수정 불가 (disabled) | - |

#### 2.3.2 코드값 (`code`)

| 검증 시점 | 조건 | 결과 | 에러 메시지 |
|:---|:---|:---|:---|
| onChange | 빈 문자열 | 에러 표시 | 코드값을 입력해주세요. |
| onChange | 소문자 입력 | 자동 대문자 변환 | - |
| onBlur | 동일 그룹 내 코드값 중복 | 에러 표시 | 동일 그룹 내에 같은 코드값이 존재합니다. |
| onChange | 한글/특수문자 포함 | 에러 표시 | 영문 대문자, 숫자, _만 사용 가능합니다. |

#### 2.3.3 그룹명/코드명 (`groupName`, `name`)

| 검증 시점 | 조건 | 결과 | 에러 메시지 |
|:---|:---|:---|:---|
| onBlur | 빈 문자열 | 에러 표시 | 그룹명/코드명을 입력해주세요. |
| onBlur | 공백만 입력 | 에러 표시 (trim 후 빈 값) | 그룹명/코드명을 입력해주세요. |
| onChange | 50자 초과 | 입력 차단 | - |
| onBlur | XSS 문자 (`<`, `>`, `"`, `'`) | 에러 표시 | 허용되지 않는 문자가 포함되어 있습니다. |

### 2.4 복합 Validation (Cross-Field)

| 조건 | 검증 규칙 | 에러 메시지 |
|:---|:---|:---|
| 그룹 비활성(active=false) 시 | 하위 코드 전체 비활성 경고 | 그룹을 비활성화하면 하위 코드도 조회되지 않습니다. 계속하시겠습니까? |
| 그룹 삭제 시 | 하위 코드가 있으면 경고 | 하위 코드도 함께 삭제됩니다. |
| 코드 정렬순서 중복 | 같은 그룹 내 동일 sortOrder | 경고만 표시 (차단하지 않음) |

## 3. 기능 상세 구현 로직 (Functional Implementation Logic)

### 3.1 이벤트 및 핸들러 명세

| 대상 | 이벤트 | 동작 및 상세 로직 | 연동 API |
|:---|:---|:---|:---|
| 페이지 진입 | Mount | 코드 그룹 목록 조회 | `GET /api/v1/codes` |
| 그룹 선택 | Click | 선택된 그룹 정보 우측 패널에 바인딩 + 코드 상세 테이블 표시 | - |
| 그룹 선택 해제 | 동일 그룹 재클릭 | 선택 해제 → 빈 상태 표시 | - |
| 그룹 추가 버튼 | Click | 우측 패널 → 생성 모드 전환, 그룹코드 필드 자동 포커싱 | - |
| 생성 > 그룹 추가 | Click | validation → API 호출 → 성공 시 생성 모드 해제 + 목록 새로고침 + 토스트 | `POST /api/v1/codes` |
| 생성 > 취소 | Click | 생성 모드 해제 | - |
| 생성 > Enter 키 | KeyDown | 그룹 추가와 동일 | - |
| 상세 > 저장 | Click | validation → 그룹 수정 API → 성공 시 목록 새로고침 + 토스트 | `PUT /api/v1/codes/{id}` |
| 상세 > 삭제 | Click | confirm → 삭제 API → 성공 시 선택 초기화 + 목록 새로고침 + 토스트 | `DELETE /api/v1/codes/{id}` |
| 상세 > 사용여부 토글 | Click | 로컬 상태 변경 (저장 시 반영) | - |
| 코드 상세 > 코드 추가 | Click | 인라인 추가 행 표시, 코드값 필드 자동 포커싱 | - |
| 코드 상세 > 추가 확인 | Click/Enter | validation → 코드 추가 API → 테이블 새로고침 + 토스트 | `POST /api/v1/codes/{groupId}/details` |
| 코드 상세 > 편집 | Click | 해당 행 인라인 편집 모드 전환 | - |
| 코드 상세 > 편집 저장 | Click/Enter | validation → 코드 수정 API → 테이블 새로고침 + 토스트 | `PUT /api/v1/codes/{groupId}/details/{detailId}` |
| 코드 상세 > 편집 취소 | Click/Escape | 인라인 편집 취소, 원본 복원 | - |
| 코드 상세 > 삭제 | Click | confirm → 코드 삭제 API → 테이블 새로고침 + 토스트 | `DELETE /api/v1/codes/{groupId}/details/{detailId}` |
| 코드 상세 > 사용여부 토글 | Click | 즉시 API 호출 (낙관적 업데이트) | `PUT /api/v1/codes/{groupId}/details/{detailId}` |
| 편집 중 다른 그룹 선택 | Click | isDirty 확인 → 미저장 confirm | - |

### 3.2 화면 상태 변화 (State Transition)

**전역 상태** (useCodeStore):
| 상태명 | 타입 | 초기값 | 설명 |
|:---|:---|:---|:---|
| `groups` | `CodeGroup[]` | `[]` | 전체 코드 그룹 목록 (상세 포함) |
| `selectedGroup` | `CodeGroup \| null` | `null` | 선택된 그룹 |
| `isLoading` | `boolean` | `false` | 로딩 상태 |
| `isDirty` | `boolean` | `false` | 변경사항 존재 여부 |
| `isCreating` | `boolean` | `false` | 생성 모드 여부 |

**지역 상태** (CodeDetail):
| 상태명 | 타입 | 설명 |
|:---|:---|:---|
| `groupName` | `string` | 편집 중인 그룹명 |
| `description` | `string` | 편집 중인 설명 |
| `active` | `boolean` | 편집 중인 사용여부 |
| `isDirty` | `boolean` | 원본 대비 변경 여부 |
| `isSubmitting` | `boolean` | 제출 중 (중복 클릭 방지) |

**지역 상태** (CodeDetailTable):
| 상태명 | 타입 | 설명 |
|:---|:---|:---|
| `isAdding` | `boolean` | 인라인 추가 행 표시 여부 |
| `newCode` | `string` | 추가 중인 코드값 |
| `newName` | `string` | 추가 중인 코드명 |
| `newSort` | `number` | 추가 중인 정렬순서 |
| `editingId` | `number \| null` | 편집 중인 코드 ID |
| `editCode` | `string` | 편집 중인 코드값 |
| `editName` | `string` | 편집 중인 코드명 |
| `editSort` | `number` | 편집 중인 정렬순서 |

**상태 전이 흐름**:
1. 페이지 진입 → `isLoading = true` → `fetchGroups()` → `isLoading = false`
2. 그룹 없음 → Empty UI (Database 아이콘 + "등록된 코드 그룹이 없습니다")
3. 그룹 선택 → `selectedGroup` 설정 → 우측 패널 렌더링 (그룹 정보 + 코드 테이블)
4. 그룹 추가 클릭 → `isCreating = true` → 우측 패널 생성 폼
5. 그룹 추가 성공 → 토스트 + 목록 새로고침 + 생성 모드 해제
6. 그룹 저장 성공 → 토스트 + 목록 새로고침 + isDirty 해제
7. 그룹 삭제 성공 → 토스트 + 목록 새로고침 + selectedGroup 초기화
8. 코드 추가/편집/삭제 → 토스트 + 목록 새로고침 (선택 그룹 유지)

### 3.3 API 엔드포인트 명세

| Method | URL | 설명 | Request Body | Response |
|:---|:---|:---|:---|:---|
| GET | `/api/v1/codes` | 전체 그룹 + 상세 목록 조회 | - | `CodeGroup[]` |
| GET | `/api/v1/codes/{id}` | 그룹 단건 조회 | - | `CodeGroup` |
| POST | `/api/v1/codes` | 그룹 생성 | `{ groupCode, groupName, description }` | `CodeGroup` |
| PUT | `/api/v1/codes/{id}` | 그룹 수정 | `{ groupName, description, active }` | `CodeGroup` |
| DELETE | `/api/v1/codes/{id}` | 그룹 삭제 (하위 코드 포함) | - | `void` |
| POST | `/api/v1/codes/{groupId}/details` | 코드 추가 | `{ code, name, sortOrder, active }` | `CodeDetail` |
| PUT | `/api/v1/codes/{groupId}/details/{detailId}` | 코드 수정 | `{ code, name, sortOrder, active }` | `CodeDetail` |
| DELETE | `/api/v1/codes/{groupId}/details/{detailId}` | 코드 삭제 | - | `void` |

### 3.4 API 에러 핸들링

| HTTP 상태 | 상황 | FE 처리 |
|:---|:---|:---|
| 400 | Validation 실패 / 잘못된 요청 | 에러 토스트 (응답 message 표시) |
| 401 | 인증 만료 | 로그인 페이지 리다이렉트 (api 인터셉터) |
| 403 | 권한 없음 | 에러 토스트 "접근 권한이 없습니다." |
| 404 | 존재하지 않는 그룹/코드 | 에러 토스트 + 목록 새로고침 |
| 409 | 중복 (그룹코드 또는 코드값) | 에러 토스트 (응답 message 표시) |
| 500 | 서버 오류 | 에러 토스트 "서버 오류가 발생했습니다." |

## 4. 예외 및 보안 정책 (Special Policies)

| 정책 | 상세 |
|:---|:---|
| **그룹코드 수정 불가** | 생성 후 groupCode는 변경 불가 (disabled 표시) |
| **그룹 삭제 시 하위 연쇄 삭제** | confirm에 "하위 코드도 함께 삭제됩니다" 경고 포함 |
| **코드값 자동 대문자** | 입력 시 자동 toUpperCase() 변환 |
| **중복 클릭 방지** | 저장/추가/삭제 시 isSubmitting 동안 버튼 disabled |
| **XSS 방지** | 그룹명/코드명에 `<`, `>`, `"`, `'` 입력 차단 |
| **미저장 데이터 보호** | 편집 중 그룹 전환 시 confirm |
| **Role-Based UI** | SUPER_ADMIN 역할만 접근 가능 |
