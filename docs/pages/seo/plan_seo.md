# [SEO 관리] 단위 화면 상세 기획서 (Detailed Functional Spec)

## 1. 개요 및 비즈니스 맥락 (Background & Goals)

- **Problem Statement (해결 과제)**:
  FrontOffice 페이지의 검색엔진 노출 품질이 개발 배포 시점에만 설정 가능하여, 운영자가 실시간으로 메타 정보를 조정할 수 없음. 페이지 메이커로 생성한 페이지도 SEO 정보가 자동으로 붙지 않아 검색 노출이 누락되는 문제가 있음.

- **Business Objective (목표)**:
  BO 관리자가 코드 배포 없이 FrontOffice 각 URL의 SEO 정보(제목·설명·OG·robots 등)를 즉시 등록·수정하고, 페이지 메이커 템플릿 생성 시 SEO 엔트리가 자동 생성되어 누락을 방지한다.

- **Success Metric (성공 지표)**:
  - 페이지 메이커 템플릿 생성 → SEO 엔트리 자동 생성률 100%
  - 운영자 SEO 정보 수정 반영 시간: 배포 없이 즉시(실시간)
  - SEO 정보 미등록 FrontOffice 페이지 0건

## 2. 정보 구조 및 메뉴 설계 (IA & Menu Planning)

- **모듈 내 전체 화면 목록**:
  - SEO 목록 (Main List) — URL 기준 등록된 SEO 정보 전체 조회

  - 자동 생성 현황 — 페이지 메이커 템플릿과 SEO 연동 상태 표시

- **메뉴 경로 (Menu Path)**: 사이트 설정 → SEO 관리

- **사용자 권한 정책**:
  - **SUPER_ADMIN**: 전체 SEO 정보 조회·등록·수정·삭제
  - **EDITOR**: 조회만 가능 (등록·수정·삭제 불가)

## 3. 화면 상세 명세 및 핵심 기능 (Detailed Screen Specs)

### Layout Structure
- 상단: 페이지 타이틀 + "등록" 버튼
- 중단: 검색바 (URL / 제목 키워드 검색)
- 하단: SEO 목록 테이블 + 페이지네이션
- 팝업: 등록·수정용 레이어 팝업 (우측 드로어 또는 중앙 팝업)

### Indispensable Feature List (필수 기능)

- **SEO 목록 조회**: URL / 제목 / 설명 / 적용여부 / 자동생성 여부 컬럼 표시
- **SEO 등록**: 관리자가 URL을 직접 입력하고 SEO 항목 작성
- **SEO 수정**: 기존 엔트리의 항목 수정 후 즉시 FrontOffice에 반영
- **SEO 삭제**: 엔트리 삭제 (FrontOffice에서 해당 URL SEO 정보 제거)
- **페이지 메이커 자동 생성**: 페이지 메이커에서 템플릿 "생성" 버튼 클릭 시 해당 slug URL로 SEO 엔트리 자동 생성 (title = 페이지명, 나머지는 빈값으로 초기값 세팅)
- **FrontOffice API 제공**: FrontOffice가 URL 기준으로 SEO 정보를 조회하는 공개 API

### Input Fields (SEO 등록·수정 팝업)

| 필드명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| URL | 텍스트 | 필수 | 적용 대상 경로 (예: `/`, `/about`) |
| 제목 (title) | 텍스트 | 필수 | `<title>` 태그 (최대 60자 권장) |
| 설명 (description) | 텍스트에어리어 | 선택 | `<meta name="description">` (최대 160자 권장) |
| 키워드 (keywords) | 텍스트 | 선택 | `<meta name="keywords">` (쉼표 구분) |
| OG 제목 | 텍스트 | 선택 | `<meta property="og:title">` (미입력 시 title과 동일) |
| OG 설명 | 텍스트에어리어 | 선택 | `<meta property="og:description">` |
| OG 이미지 URL | 텍스트 | 선택 | `<meta property="og:image">` |
| robots | 선택(select) | 선택 | `index,follow` / `noindex,nofollow` / `noindex,follow` / `nofollow` |
| canonical URL | 텍스트 | 선택 | 중복 URL 정규화용 |
| 적용 여부 | 토글 | 필수 | 비활성화 시 FrontOffice에 미반영 |

## 4. [Proactive Update] 혁신 및 개선 제안 (Enhancement Suggestions)

- **아이디어 1 — 글자수 실시간 카운터**: title(60자), description(160자) 입력 시 남은 글자 수 표시
  - 기대 효과: 검색 결과 잘림 현상 사전 방지

- **아이디어 2 — 미적용 페이지 일괄 알림**: 페이지 메이커로 생성된 템플릿 중 SEO 제목이 비어있는 항목 강조 표시
  - 기대 효과: SEO 누락 페이지를 운영자가 한눈에 파악 가능

- **아이디어 3 — OG 미리보기**: 저장 전 카카오톡·슬랙 링크 공유 시 표시될 모습 미리보기 (제목·설명·이미지 카드 형태)
  - 기대 효과: 비개발자 운영자도 공유 화면 결과를 직관적으로 확인

## 5. 비즈니스 로직 및 예외 정책 (Logic & Exception Policy)

### 상세 비즈니스 정책

- **URL 중복 금지**: 동일 URL로 SEO 엔트리 중복 등록 불가. 등록 시도 시 "이미 등록된 URL입니다" 안내 후 기존 엔트리 수정 유도
- **자동 생성 엔트리**: 페이지 메이커 "생성" 시 자동 생성된 엔트리는 `autoGenerated = true` 표시. 템플릿 삭제 시 연관 SEO 엔트리도 함께 삭제
- **OG 필드 미입력 fallback**: ogTitle 미입력 → title 값으로 대체, ogDescription 미입력 → description 값으로 대체하여 API 응답
- **적용 여부 = 비활성**: FrontOffice API 응답에서 해당 URL 제외 (404가 아닌 빈 응답)
- **FrontOffice 캐싱 정책**: SEO 정보는 FrontOffice에서 요청 시마다 실시간 조회 (BO 수정 즉시 반영)

### 입력 검증 정책

- URL: `/`로 시작, 영문·숫자·하이픈·슬래시·언더스코어만 허용
- title: 1~100자
- description: 최대 500자
- OG 이미지 URL: http/https로 시작하는 유효한 URL 형식

### 예외 처리

- FrontOffice가 미등록 URL로 API 요청 시: `null` 응답 (FrontOffice는 기본 메타값 사용)
- 네트워크 오류: 토스트 메시지 표시, 기존 데이터 유지
- 빈 목록: "등록된 SEO 정보가 없습니다" Empty State 표시

## 6. 공통 모듈 및 자산화 전략 (Commonality & Assets)

- **공통 UI**: 기존 `[slug]/page.tsx` 동적 렌더러 패턴(검색 + 테이블 + 레이어팝업) 동일하게 적용
- **공통 로직**: 페이지 메이커 템플릿 생성(`PageTemplateService`) 시 SEO 엔트리 자동 생성 훅 추가
- **FE 상태 관리**: 기존 Zustand 스토어 패턴 동일하게 적용

## 7. 디자인/개발 핸드오버 체크리스트 (Step 2 & 3 Support)

- **디자인 필수 표현**:
  - title/description 글자수 카운터 (권장 글자수 초과 시 주황색 경고)
  - 자동생성 엔트리 배지 표시 (`자동` 뱃지)
  - 적용여부 토글 (활성=파란색, 비활성=회색)
  - 미작성 필수항목(제목) 강조 표시

- **기능적 인터랙션**:
  - 목록에서 행 클릭 → 수정 팝업 오픈
  - 등록/수정 저장 성공 → 토스트 메시지 + 목록 자동 새로고침
  - 삭제 → 확인 다이얼로그 후 처리

- **성공 인수 조건 (Acceptance Criteria)**:
  1. 페이지 메이커에서 "생성" 클릭 시 SEO 목록에 해당 URL 엔트리가 자동으로 나타난다
  2. BO에서 SEO 정보 수정 후 FrontOffice URL 접근 시 수정된 `<title>`, `<meta>` 태그가 즉시 반영된다
  3. URL 중복 등록 시 저장이 거부되고 오류 메시지가 표시된다
  4. 적용 여부를 비활성화하면 FrontOffice API에서 해당 URL SEO 정보가 반환되지 않는다
  5. 미등록 URL로 FrontOffice가 API 요청 시 null을 반환하며 FrontOffice는 정상 동작한다
