## 2. 역할 카드 컴포넌트

```
┌────────────────────────────────────────────┐
│  상단: 색상 dot + 역할 코드 + 시스템 뱃지    │
│  ──────────────────────────────────────    │
│  표시명 (bold, 14px)                        │
│  설명 (gray, 12px, 2줄 clamp)              │
│  ──────────────────────────────────────    │
│  하단: 사용 인원 뱃지     [수정] [삭제]      │
└────────────────────────────────────────────┘
```

| 요소 | 스펙 |
|:---|:---|
| 카드 배경 | `#ffffff`, border `#e8eaed`, radius `12px` |
| 색상 dot | `10px` 원, 역할 color 값 |
| 역할 코드 | `font-mono`, `11px`, `#6b7280` |
| 시스템 뱃지 | `🔒` + "시스템", bg `#f4f5f7`, text `#6b7280`, `11px` |
| 표시명 | `14px`, `font-semibold`, `#111827` |
| 설명 | `12px`, `#9ca3af`, `line-clamp-2` |
| 사용 인원 | `N명`, bg `#eef0fd`, text `#4361ee`, `11px` |
| 수정 버튼 | icon `Edit2`, `w-7 h-7`, hover `#eef0fd` / `#4361ee` |
| 삭제 버튼 | icon `Trash2`, `w-7 h-7`, hover `#fef2f2` / `#ef4444`, is_system이면 `opacity-30 cursor-not-allowed` |

## 3. Drawer 컴포넌트

| 요소 | 스펙 |
|:---|:---|
| 너비 | `420px` (사용자 관리 Drawer와 동일) |
| 제목 | 신규: "역할 추가" / 수정: "역할 수정" |
| 부제 | 신규: "새로운 역할을 등록합니다" / 수정: "역할 정보를 수정합니다" |

### 역할 코드 필드
- 신규: 입력 가능, 자동 대문자 변환
- 수정: `readonly` + 배경 `#f9fafb`, 잠금 아이콘 표시 + 안내문 "역할 코드는 변경할 수 없습니다"

### 색상 프리셋 (8색)
```
#4361ee  #10b981  #f59e0b  #ef4444
#8b5cf6  #06b6d4  #ec4899  #6b7280
```
- 선택된 색상: `ring-2 ring-offset-1 ring-[selected-color]` 표시

---

## 4. 색상 뱃지 (역할 코드 배지)

사용자 관리 페이지의 역할 배지도 Role 테이블의 color 값으로 동적 렌더링되도록 변경.

| before | after |
|:---|:---|
| 하드코딩 `SUPER_ADMIN → #4361ee` | Role API 응답의 `color` 사용 |
| 하드코딩 `EDITOR → #6b7280` | 동적 color 매핑 |

## 5. 디자인 토큰 (기존 시스템과 통일)

| Token | Value | 사용처 |
|:---|:---|:---|
| `color-surface-card` | `#ffffff` | 카드 배경 |
| `color-border-default` | `#e8eaed` | 카드/드로어 테두리 |
| `color-text-primary` | `#111827` | 표시명 |
| `color-text-subtle` | `#9ca3af` | 설명, 코드 |
| `color-accent` | `#4361ee` | 버튼, 인원 뱃지 |
| `color-page-bg` | `#f4f5f7` | 페이지 배경 |
