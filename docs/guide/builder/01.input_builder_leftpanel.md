# 빌더 좌측 패널 (Input Left Panel) 개발 가이드

> 작성일: 2026-05-03 / 최종 수정: 2026-05-06  
> 대상: 빌더 페이지 좌측 설정 패널을 새로 만들거나 수정하는 모든 에이전트  
> 관련 파일: `make/quick-list/page.tsx`, `make/quick-detail/page.tsx`, `make/widget/page.tsx`

---

## 1. 개요

빌더 페이지는 **좌측 설정 패널(Input)** + **우측 미리보기(Output)** 구조로 나뉜다.  
이 문서는 **좌측 설정 패널**만 다룬다.

```
<div className="grid grid-cols-[340px_1fr] gap-5 items-start">
  ┌──────────────────────┐   ┌──────────────────────────────┐
  │  좌측 Input (340px)   │   │   우측 Output (미리보기)       │
  │  ← 이 문서 대상        │   │   PageGridRenderer 담당       │
  └──────────────────────┘   └──────────────────────────────┘
```

---

## 2. 좌측 패널 고정 구조

모든 빌더 페이지의 좌측 패널은 **아래 4-레이어 구조**를 반드시 따른다.

```
<div className="bg-white border border-slate-200 rounded-xl sticky top-4">

  [Layer 1] <TemplateLoader>                  ← useTemplateManagement 훅과 연동
  ─────────────────────────────────
  [Layer 2] <OutputModePanel> (선택)           ← page/layerpopup 전환이 필요한 경우만
  ─────────────────────────────────
  [Layer 3] 위젯 셀 영역
            └── 위젯 헤더 (bg-slate-900)
            └── 컨텐츠 행 반복
                └── ContentRowHeader          ← 헤더 클릭으로 편집 토글
                └── SizeSettingPanel          ← 편집 중일 때만 표시
                └── CommonBuilderDispatcher   ← 편집 중일 때만 표시

</div>
```

## 3. 빌더 페이지 유형

### 3-1. 고정 구조 (quick-list, quick-detail)

위젯 구성이 고정되어 있고 추가/삭제/재정렬이 불가능한 빌더.

| 페이지 | 고정 위젯 | 파일 |
|---|---|---|
| quick-list | Search + Space + Table | `make/quick-list/page.tsx` |
| quick-detail | Form + Space | `make/quick-detail/page.tsx` |

**상태 패턴**:
```typescript
// 각 위젯을 개별 state로 관리
const [searchContent, setSearchContent] = useState<FixedContentItem<SearchWidget>>(createSearchContent);
const [spaceContent,  setSpaceContent]  = useState<FixedContentItem<SpaceWidget>>(createSpaceContent);
const [tableContent,  setTableContent]  = useState<FixedContentItem<TableWidget>>(createTableContent);

// 편집 중인 컨텐츠 ID
const [editingContentId, setEditingContentId] = useState<string | null>(null);
```

**FixedContentItem 타입**:
```typescript
interface FixedContentItem<W> {
    id: string;      // 'fixed-search', 'fixed-form' 등 고유 ID
    colSpan: number; // 1~12
    rowSpan: number; // 1~20
    widget: W;       // 위젯 데이터
}
```

### 3-2. 동적 구조 (widget)

위젯 추가/삭제/재정렬이 자유로운 빌더.

**상태 패턴**:
```typescript
// 전체 위젯 배열로 관리
const [widgetItems, setWidgetItems] = useState<PageWidgetItem[]>([]);
const [editingItemId,    setEditingItemId]    = useState<string | null>(null);
const [editingContentId, setEditingContentId] = useState<string | null>(null);
```

## 4. 핵심 컴포넌트

### 4-1. CommonBuilderDispatcher ⭐ 가장 중요

**모든 빌더 페이지에서 위젯 빌더를 렌더링할 때 반드시 이 컴포넌트를 경유해야 한다.**

```
❌ 절대 금지 — 개별 빌더 직접 사용
<SearchWidgetBuilder widget={...} onChange={...} />
<SpaceBuilder widget={...} onChange={...} />
<TableBuilder widget={...} onChange={...} />
<FormBuilder widget={...} onChange={...} />

✅ 올바른 방법 — 항상 CommonBuilderDispatcher 경유
<CommonBuilderDispatcher
    widget={content.widget}
    onChange={w => setContent(prev => ({ ...prev, widget: w as WidgetType }))}
    context={{ ... }}
/>
```

**파일 경로**: `make/_shared/components/builder/CommonBuilderDispatcher.tsx`

#### context Props 전체 명세

```typescript
context: {
    // ✅ 필수 — Form, Table, Category의 DB Slug 선택 드롭다운에 사용
    slugOptions: { id: number; slug: string; name: string }[];

    // ✅ 권장 — Space ActionButton의 팝업 연결, Category 등록/수정 팝업에 사용
    pageTemplates?: TemplateItem[];

    // Table 위젯에서 연결할 Search 목록 (Table이 있을 때 필수)
    searchWidgets?: Array<{ widgetId: string; contentKey: string }>;

    // Space ActionButton에서 연결할 Form 목록 (Form + Space 조합일 때 필수)
    formWidgets?: Array<{ widgetId: string; contentKey: string; connectedSlug?: string }>;

    // Form 필드 colSpan 최대값 (기본 12, 우측 드로어 모드는 2)
    maxColSpan?: number;

    // Category 위젯에서 parentWidgetId 선택에 사용
    categoryWidgets?: { widgetId: string; label?: string; depth: number }[];

    // ⚠️ Space에서 ActionButton만 추가 가능하게 제한 (quick-list, quick-detail 전용)
    actionButtonOnly?: boolean;
}
```

#### 위젯 타입별 context 설정 가이드

| 위젯 타입 | 필수 context | 선택 context |
|---|---|---|
| `search` | _(없음)_ | `pageTemplates` |
| `table` | `slugOptions`, `searchWidgets` | `pageTemplates` |
| `form` | `slugOptions` | `pageTemplates`, `maxColSpan` |
| `space` | `pageTemplates` | `formWidgets`, `actionButtonOnly`, `maxColSpan` |
| `category` | `slugOptions`, `pageTemplates` | `categoryWidgets` |
| `sublist` | _(없음)_ | _(없음)_ |

#### actionButtonOnly 규칙

```typescript
// quick-list, quick-detail의 Space: 버튼만 추가 가능
context={{ ..., actionButtonOnly: true }}

// widget 빌더의 Space: 텍스트(textarea) + 버튼 둘 다 가능
context={{ ... }}  // actionButtonOnly 생략
```

> **왜?** quick-list/quick-detail의 공간영역은 등록·저장·닫기 등 액션 버튼 전용 영역이다.  
> widget 빌더는 자유 구성이므로 텍스트도 허용한다.

### 4-2. ContentRowHeader

컨텐츠 행 상단에 표시되는 헤더. 클릭 시 빌더 설정 패널이 토글된다.

**파일 경로**: `make/_shared/components/builder/ContentRowHeader.tsx`

```typescript
<ContentRowHeader
    widgetType="form"           // 'search'|'table'|'form'|'space'|'category'|'sublist'
    label="Form — myKey"        // 헤더에 표시할 라벨 (contentKey 포함 권장)
    colSpan={content.colSpan}
    rowSpan={content.rowSpan}
    isEditing={editingContentId === content.id}  // 현재 편집 중 여부
    isFixed                     // 고정 구조: isFixed 사용 (삭제 버튼 없음)
    onToggle={() => setEditingContentId(
        editingContentId === content.id ? null : content.id
    )}
    // 동적 구조에서만 사용:
    // onRemove={() => removeContent(itemId, contentId)}
    // dragHandleProps={handleProps}  // DnD 드래그 핸들
/>
```

**라벨 작성 규칙**:
```typescript
// contentKey가 있으면 포함, 없으면 타입명만
label={`검색${searchContent.widget.contentKey ? ` — ${searchContent.widget.contentKey}` : ''}`}
label={`Form${formWidget.contentKey ? ` — ${formWidget.contentKey}` : ''}`}
label="공간영역"   // space는 contentKey 없음
label="데이터테이블"  // table도 contentKey 없음
```

**지원 위젯 타입 색상**:
| widgetType | 색상 |
|---|---|
| `search` | 파란색 (blue-700) |
| `table` | 초록색 (emerald-700) |
| `form` | 보라색 (violet-700) |
| `space` | 황색 (amber-700) |
| `category` | 청록색 (cyan-700) |
| `sublist` | 남색 (indigo-700) |

### 4-3. SizeSettingPanel

컨텐츠 행의 colSpan/rowSpan을 숫자 입력으로 변경하는 패널.  
**항상 ContentRowHeader 바로 아래, CommonBuilderDispatcher 바로 위에 배치한다.**

**파일 경로**: `make/_shared/components/builder/SizeSettingPanel.tsx`

```typescript
<SizeSettingPanel
    colSpan={content.colSpan}
    rowSpan={content.rowSpan}
    maxColSpan={om.isRightDrawer ? 2 : 12}  // 우측 드로어 모드는 2 제한
    // maxRowSpan={20}  // 기본값 20, 변경 필요 시만 명시
    onColSpanChange={v => updateSize(v, content.rowSpan)}
    onRowSpanChange={v => updateSize(content.colSpan, v)}
/>
```

**updateSize 패턴 (고정 구조)**:
```typescript
// colSpan/rowSpan 범위 제한 포함
const updateSize = <W,>(
    setter: React.Dispatch<React.SetStateAction<FixedContentItem<W>>>,
    colSpan: number,
    rowSpan: number,
) => setter(prev => ({
    ...prev,
    colSpan: Math.max(1, Math.min(maxCol, colSpan)),
    rowSpan: Math.max(1, rowSpan),
}));
```

### 4-4. OutputModePanel

page/layerpopup 출력 모드 전환 탭. 팝업 타입/제목/크기 설정 포함.  
**quick-list는 항상 page 모드라 생략. quick-detail, widget에서 사용.**

**파일 경로**: `make/_shared/components/builder/OutputModePanel.tsx`

```typescript
// useOutputMode 훅과 함께 사용
const om = useOutputMode();

<OutputModePanel
    outputMode={om.outputMode}
    layerType={om.layerType}
    layerTitle={om.layerTitle}
    layerWidth={om.layerWidth}
    onOutputModeChange={om.setOutputMode}
    onLayerTypeChange={om.setLayerType}
    onLayerTitleChange={om.setLayerTitle}
    onLayerWidthChange={om.setLayerWidth}
/>
```

**우측 드로어 모드에서 colSpan 자동 클램핑 필수**:
```typescript
// layerType이 'right'로 변경될 때 Form + Space 양쪽 모두 클램핑
useEffect(() => {
    if (om.outputMode === 'layerpopup' && om.layerType === 'right') {
        setFormContent(prev => ({
            ...prev,
            colSpan: Math.min(prev.colSpan, 2),
            widget: {
                ...prev.widget,
                fields: (prev.widget as FormWidget).fields.map(
                    f => ({ ...f, colSpan: Math.min(f.colSpan, 2) })
                ),
            },
        }));
        setSpaceContent(prev => ({
            ...prev,
            colSpan: Math.min(prev.colSpan, 2),
            widget: {
                ...prev.widget,
                items: (prev.widget as SpaceWidget).items.map(
                    i => ({ ...i, colSpan: Math.min(i.colSpan ?? 1, 2) as 1|2|3|4|5 })
                ),
            },
        }));
    }
}, [om.layerType, om.outputMode]);
```

> widget 빌더는 widgetItems 배열 전체를 순회하여 form/space 내부까지 일괄 클램핑한다.  
> 자세한 패턴은 `make/widget/page.tsx` 참고.

### 4-5. useTemplateManagement ⭐ 불러오기·저장 상태 통합 훅

**불러오기(목록/삭제/복사) + 저장(모달/확인) 상태와 핸들러를 하나의 훅으로 제공한다.**  
모든 빌더 페이지(quick-list, quick-detail, widget)에서 동일하게 사용한다.

**파일 경로**: `make/_shared/hooks/useTemplateManagement.ts`

```typescript
// templateType: 'PAGE' | 'QUICK_LIST' | 'QUICK_DETAIL'
const tm = useTemplateManagement('QUICK_LIST');
```

**훅이 반환하는 값 전체 목록**:

| 반환값 | 설명 |
|---|---|
| `filteredTemplates` | 검색어 기준 필터된 목록 (파생) |
| `isLoadingList` | 목록 로딩 중 여부 |
| `showLoadDropdown` / `setShowLoadDropdown` | 드롭다운 열림 상태 |
| `loadSearch` / `setLoadSearch` | 검색어 상태 |
| `isDeletingId` / `isDuplicatingId` | 삭제/복사 처리 중인 ID |
| `loadTemplateList` | 목록 조회 함수 |
| `onLoadSuccess(tpl)` | 불러오기 성공 후 공통 처리 (ID 업데이트 + 드롭다운 닫기 + toast) |
| `handleDeleteTemplate(id)` | 삭제 처리 |
| `handleDuplicateTemplate(tpl)` | 복사 처리 |
| `currentTemplateId` / `currentTemplateName` | 현재 불러온 템플릿 정보 |
| `showSaveModal` / `setShowSaveModal` | 저장 모달 열림 상태 |
| `saveModalName` / `setSaveModalName` | 저장 모달 이름 |
| `saveModalSlug` / `setSaveModalSlug` | 저장 모달 slug |
| `saveModalDesc` / `setSaveModalDesc` | 저장 모달 설명 |
| `isSaving` | 저장 중 여부 |
| `openSaveModal()` | 저장 모달 열기 (currentTemplateName 자동 세팅) |
| `handleSaveConfirm(widgetItems, extra?)` | 저장 확인 (saveTemplate API 호출) |

**handleLoadSelect 패턴** — 페이지 고유 파싱 후 반드시 `tm.onLoadSuccess(tpl)` 마지막에 호출:
```typescript
const handleLoadSelect = (tpl: TemplateItem) => {
    try {
        const config = JSON.parse(tpl.configJson);
        // 페이지별 고유 파싱 (각 페이지에서 직접 작성)
        setSearchContent(config.searchContent || createSearchContent());
        setEditingContentId(null);
        tm.onLoadSuccess(tpl); // ← 반드시 마지막에 호출 (ID/Name 업데이트 + 드롭다운 닫기 + toast)
    } catch {
        import('sonner').then(({ toast }) => toast.error('설정 파일 파싱에 실패했습니다.'));
    }
};
```

**저장 연동 패턴**:
```typescript
// 저장 버튼 — validation 통과 후 tm.openSaveModal() 호출
<button onClick={tm.openSaveModal}>저장</button>

// SaveModal 연동
<SaveModal
    show={tm.showSaveModal}
    onClose={() => tm.setShowSaveModal(false)}
    isEdit={!!tm.currentTemplateId}
    name={tm.saveModalName}    slug={tm.saveModalSlug}    desc={tm.saveModalDesc}
    isSaving={tm.isSaving}
    onNameChange={tm.setSaveModalName}
    onSlugChange={tm.setSaveModalSlug}
    onDescChange={tm.setSaveModalDesc}
    onConfirm={() => tm.handleSaveConfirm(buildWidgetItems(), { outputMode: om.outputMode })}
    toSlug={toSlug}
/>
```

### 4-6. TemplateLoader

불러오기 드롭다운 UI 컴포넌트. `useTemplateManagement` 훅과 항상 함께 사용한다.  
**직접 드롭다운 UI를 구현하지 않는다. 반드시 이 컴포넌트를 사용한다.**

**파일 경로**: `make/_shared/components/builder/TemplateLoader.tsx`

```tsx
const tm = useTemplateManagement('QUICK_LIST');

<TemplateLoader
    {...tm}   // isLoadingList, showLoadDropdown, loadSearch, filteredTemplates 등 일괄 전달
    onToggle={() => { tm.setShowLoadDropdown(v => !v); if (!tm.showLoadDropdown) tm.loadTemplateList(); }}
    onSearchChange={tm.setLoadSearch}
    onSelect={handleLoadSelect}          // ← 페이지별 고유 파싱 함수
    onDelete={tm.handleDeleteTemplate}
    onDuplicate={tm.handleDuplicateTemplate}
/>
```

> `{...tm}`으로 스프레드하면 TemplateLoader가 필요로 하는 상태가 한 번에 전달된다.  
> `onSelect`만 페이지마다 고유한 파싱 로직이 있으므로 직접 작성한다.

## 5. 완성된 컨텐츠 행 패턴 (표준 코드)

### 5-1. 고정 구조 컨텐츠 행 (search 예시)

```tsx
{/* ── 검색 컨텐츠 행 ── */}
<div className="border-t border-slate-100">
    <ContentRowHeader
        widgetType="search"
        label={`검색${searchContent.widget.contentKey ? ` — ${searchContent.widget.contentKey}` : ''}`}
        colSpan={searchContent.colSpan}
        rowSpan={searchContent.rowSpan}
        isEditing={editingContentId === 'fixed-search'}
        isFixed
        onToggle={() => setEditingContentId(
            editingContentId === 'fixed-search' ? null : 'fixed-search'
        )}
    />
    {editingContentId === 'fixed-search' && (
        <div className="border-t border-slate-100 bg-slate-50/50">
            <SizeSettingPanel
                colSpan={searchContent.colSpan}
                rowSpan={searchContent.rowSpan}
                onColSpanChange={v => updateSize(setSearchContent, v, searchContent.rowSpan)}
                onRowSpanChange={v => updateSize(setSearchContent, searchContent.colSpan, v)}
            />
            <div className="px-3 pb-2 pt-1">
                <CommonBuilderDispatcher
                    widget={searchContent.widget}
                    onChange={w => setSearchContent(prev => ({
                        ...prev,
                        widget: w as typeof searchContent.widget,
                    }))}
                    context={{ slugOptions, pageTemplates }}
                />
            </div>
        </div>
    )}
</div>
```

### 5-2. 위젯 타입별 context 실전 예시

```tsx
        slugOptions,

        // actionButtonOnly 생략 — 자유 구성
    }}

{/* Category 위젯 */}
<CommonBuilderDispatcher
    widget={categoryContent.widget}
    onChange={w => setCategoryContent(prev => ({ ...prev, widget: w as CategoryWidget }))}
    context={{
        slugOptions,
```

## 6. 전체 좌측 패널 골격 코드

새 빌더 페이지를 만들 때 아래 골격을 복사해서 시작한다.

```tsx
{/* ════════════════════════════════ */}
```

## 9. 실수하기 쉬운 케이스

### 케이스 1: onChange 타입 단언 누락

CommonBuilderDispatcher의 `onChange`는 `AnyWidget`을 반환한다.  
각 state setter는 구체적인 위젯 타입을 요구하므로 반드시 타입 단언을 해야 한다.

```typescript
// ❌ 타입 오류 발생
onChange={w => setSearchContent(prev => ({ ...prev, widget: w }))}

// ✅ 타입 단언 필요
onChange={w => setSearchContent(prev => ({ ...prev, widget: w as SearchWidget }))}
onChange={w => setFormContent(prev => ({ ...prev, widget: w as FormWidget }))}
onChange={w => setTableContent(prev => ({ ...prev, widget: w as TableWidget }))}
onChange={w => setSpaceContent(prev => ({ ...prev, widget: w as SpaceWidget }))}
```

### 케이스 2: Form + Space 연동 시 formWidgets 동기화 누락

Space의 ActionButton에서 "Form 저장/삭제" 연결 기능이 동작하려면  
현재 Form 위젯 정보를 `formWidgets`로 전달해야 한다.

```typescript
// Form 위젯 참조를 변수로 캐싱
const formWidgetRef = formContent.widget as FormWidget;

// Space context에 전달
context={{
    ...,
    formWidgets: [{
        widgetId: formWidgetRef.widgetId,
        contentKey: formWidgetRef.contentKey,
        connectedSlug: formWidgetRef.connectedSlug,  // ← 이것도 필수
    }],
}}
```

### 케이스 3: 우측 드로어 모드 전환 시 colSpan 클램핑 누락

우측 드로어로 전환할 때 **Form 필드와 Space 아이템 양쪽 모두** colSpan을 줄여야 한다.

```typescript
// quick-detail: Form + Space 둘 다 클램핑
useEffect(() => {
    if (om.outputMode === 'layerpopup' && om.layerType === 'right') {
        setFormContent(prev => ({
            ...prev,
            colSpan: Math.min(prev.colSpan, 2),
            widget: {
                ...prev.widget,
                fields: (prev.widget as FormWidget).fields.map(
                    f => ({ ...f, colSpan: Math.min(f.colSpan, 2) })
                ),
            },
        }));
        setSpaceContent(prev => ({
            ...prev,
            colSpan: Math.min(prev.colSpan, 2),
            widget: {
                ...prev.widget,
                items: (prev.widget as SpaceWidget).items.map(
                    i => ({ ...i, colSpan: Math.min(i.colSpan ?? 1, 2) as 1|2|3|4|5 })
                ),
            },
        }));
    }
}, [om.layerType, om.outputMode]);
```

```typescript
// widget 빌더: widgetItems 전체 순회 — 위젯/컨텐츠/내부 필드까지 일괄 클램핑
useEffect(() => {
    if (om.outputMode === 'layerpopup' && om.layerType === 'right') {
        setWidgetItems(prev => prev.map(item => ({
            ...item,
            colSpan: Math.min(item.colSpan, 2),
            contents: item.contents.map(c => ({
                ...c,
                colSpan: Math.min(c.colSpan, 2),
                widget: c.widget.type === 'form'
                    ? { ...c.widget, fields: (c.widget as FormWidget).fields.map(f => ({ ...f, colSpan: Math.min(f.colSpan, 2) })) }
                    : c.widget.type === 'space'
                    ? { ...c.widget, items: (c.widget as SpaceWidget).items.map(i => ({ ...i, colSpan: Math.min(i.colSpan ?? 1, 2) as 1|2|3|4|5 })) }
                    : c.widget,
            })),
        })));
    }
}, [om.layerType, om.outputMode]);
```

### 케이스 4: max-h 클래스 조정 누락

좌측 패널 스크롤 영역은 Layer 2(OutputModePanel) 유무에 따라 max-h 값이 다르다.

```
OutputModePanel 없음 (quick-list): max-h-[calc(100vh-240px)]
OutputModePanel 있음 (quick-detail): max-h-[calc(100vh-320px)]
OutputModePanel 있음 (widget): max-h-[calc(100vh-280px)]
```
