# 빌더 우측 출력 패널 (Output Panel) 개발 가이드

> 작성일: 2026-05-03 / 최종 수정: 2026-05-06  
> 대상: 미리보기·렌더링 계층을 새로 만들거나 수정하는 모든 에이전트  
> 관련 파일: `renderer/RendererContainer.tsx`, `renderer/PageGridRenderer.tsx`, `renderer/WidgetRenderer.tsx`, `components/layout/PageLayout.tsx`, `components/layout/GridCell.tsx`

---

## 1. 개요

빌더 우측 출력 패널은 단순한 미리보기가 아니다.  
**빌더 템플릿 · 빌더 미리보기 · 운영 페이지 — 세 곳에서 완전히 동일한 UI가 나와야 한다.**

```
┌─────────────────────────────────────────────────────────────────┐
│   진입점 A              진입점 B              진입점 C             │
│   builder-contents      빌더 미리보기         운영 페이지          │
│   -layout               (make/*/page.tsx)     (generated/*)       │
│                                                                   │
│        ↓                      ↓                    ↓             │
│   ┌─────────────────────────────────────────────────────────┐    │
│   │        동일한 컴포넌트 스택이 조립되어 렌더링됨              │    │
│   │  PageLayout → PageGridRenderer → WidgetRenderer →       │    │
│   │  RendererContainer → FieldRenderer                      │    │
│   └─────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────┘
```

> **핵심 원칙**: UI는 각 진입점에 하드코딩하지 않는다.  
> 컴포넌트를 독립적으로 만들고, 세 진입점에서 동일하게 **조립(assemble)** 만 한다.

---

## 2. 렌더링 계층 구조 ⭐ 가장 중요

전체 계층은 아래 5단계로 구성된다. 각 레이어는 독립적인 단일 책임을 갖는다.

```
PageLayout
│  역할: 12컬럼 격자 컨테이너, 그리드 가이드라인 on/off
│  파일: bo/src/components/layout/PageLayout.tsx
│  grid: gridAutoRows=72px, rowGap=8px
│
└── PageGridRenderer  (또는 builder-contents-layout에서 GridCell 직접)
│     역할: 위젯 배열을 받아 각 위젯을 GridCell에 배치
│     파일: make/_shared/components/renderer/PageGridRenderer.tsx
│     grid: gridAutoRows=72px, rowGap=8px
│
└── GridCell (colSpan × rowSpan)
│     역할: 해당 위젯이 차지하는 격자 셀, overflow:clip
│     파일: bo/src/components/layout/GridCell.tsx
│     높이 공식: rowSpan × ROW_HEIGHT - GAP_SIZE  (예: rowSpan=1 → 72px)
│
└── WidgetRenderer
│     역할: widget.type에 따라 올바른 Renderer를 선택해 렌더링
│     파일: make/_shared/components/renderer/WidgetRenderer.tsx
│
└── RendererContainer
│     역할: 모든 Renderer 공통 최상위 컨테이너
│           테두리·배경·내부 grid 배치를 하나로 통일
│     파일: make/_shared/components/renderer/RendererContainer.tsx
│     grid: gridAutoRows=72px, rowGap=8px  (contentColSpan 전달 시)
│
└── FieldRenderer / 컨텐츠 컴포넌트
      역할: 실제 필드(입력창·셀렉트·버튼 등) 렌더링
      파일: make/_shared/components/renderer/fields/
```

## 3. 그리드 수치 기준 (절대 하드코딩 금지)

모든 높이 계산의 단일 출처(Single Source of Truth)는 `GridCell.tsx` 상수다.

```typescript
// bo/src/components/layout/GridCell.tsx
export const ROW_HEIGHT = 80;  // 행 1개의 총 높이 (track + gap 합산)
export const GAP_SIZE   = 8;   // 행 간 gap
```

이 두 상수로부터 모든 높이가 계산된다:

| 위치 | gridAutoRows | rowGap | 설명 |
|---|---|---|---|
| PageLayout | `ROW_HEIGHT - GAP_SIZE` = **72px** | `GAP_SIZE` = **8px** | 외부 페이지 격자 |
| PageGridRenderer | `ROW_HEIGHT - GAP_SIZE` = **72px** | `GAP_SIZE` = **8px** | 위젯 배치 격자 |
| GridCell 높이 | — | — | `rowSpan × ROW_HEIGHT - GAP_SIZE` |
| RendererContainer | `ROW_HEIGHT - GAP_SIZE` = **72px** | `GAP_SIZE` = **8px** | 필드 배치 내부 격자 |

```
rowSpan=1 → GridCell 높이 = 1×80 - 8 = 72px
rowSpan=2 → GridCell 높이 = 2×80 - 8 = 152px
rowSpan=3 → GridCell 높이 = 3×80 - 8 = 232px
```

> ⚠️ **절대 금지**: `80px`, `72px`, `8px`를 소스 코드에 직접 숫자로 쓰지 않는다.  
> 반드시 `ROW_HEIGHT`, `GAP_SIZE`, `ROW_HEIGHT - GAP_SIZE`를 import해서 사용한다.

## 4. 각 계층 컴포넌트 상세

### 4-1. PageLayout

12컬럼 격자를 만들어 주는 **페이지 컨테이너**.  
Tailwind `grid-cols-12`(클래스) + 인라인 스타일로 행 높이·gap을 설정한다.

```tsx
// bo/src/components/layout/PageLayout.tsx
<PageLayout mode="preview">   {/* 빌더 미리보기·템플릿 */}
  {children}                  {/* GridCell 들이 들어옴 */}
</PageLayout>

<PageLayout mode="live">      {/* 운영 페이지 */}
  {children}
</PageLayout>
```

| mode | showGrid 기본값 | 설명 |
|---|---|---|
| `preview` | `true` | 격자선·테두리·배경 표시 (g 키로 토글 가능) |
| `live` | `false` | 격자선 없음, 순수 UI만 표시 |

---

### 4-2. GridCell

PageLayout 격자 위에 놓이는 **단위 셀**.  
`colSpan`(열 점유)과 `rowSpan`(행 점유)으로 위젯 크기를 결정한다.

```tsx
// bo/src/components/layout/GridCell.tsx
<GridCell colSpan={12} rowSpan={2}>
  <WidgetRenderer mode="preview" widget={widget} contentColSpan={12} />
</GridCell>
```

```typescript
// GridCell 내부 — height 계산 공식 (직접 수정 금지)
height: `${rowSpan * ROW_HEIGHT - GAP_SIZE}px`
overflow: 'clip'   // clip: rounded border 포함, video GPU 레이어는 clip 안 함
```

> ⚠️ `overflow: 'clip'`이기 때문에 GridCell 높이를 초과하는 컨텐츠는 잘린다.  
> RendererContainer 내부 grid 높이가 GridCell을 초과하지 않도록 주의한다.

---

### 4-3. PageGridRenderer

위젯 배열 `widgetItems`를 받아 각 위젯을 **GridCell에 자동 배치**하는 렌더러.

```tsx
// make/_shared/components/renderer/PageGridRenderer.tsx
<PageGridRenderer
    mode="preview"          // 'preview' | 'live'
    widgetItems={[          // PageWidgetItem[]
        {
            id: 'wi-1',
            colSpan: 12,
            rowSpan: 6,
            contents: [     // PageContentItem[] — 1 outer 셀 안에 복수 컨텐츠 가능
                { id: 'ci-1', colSpan: 12, rowSpan: 5, widget: formWidget },
                { id: 'ci-2', colSpan: 12, rowSpan: 1, widget: spaceWidget },
            ],
        },
    ]}
/>
```

**두 계층 타입 구조:**

| 타입 | 역할 | 주요 필드 |
|---|---|---|
| `PageWidgetItem` | 외부 셀 — GridCell 하나에 대응 | `id`, `colSpan`, `rowSpan`, `contents` |
| `PageContentItem` | 내부 컨텐츠 아이템 — 위젯 1개 | `id`, `colSpan`, `rowSpan`, `widget` |

**live 모드 주요 추가 props:**

| prop | 타입 | 용도 |
|---|---|---|
| `onContentAction` | `(widgetIds: string[], action: 'save' \| 'delete') => void` | Space 버튼 → Form·SubList 저장/삭제 |
| `subListRowsMap` | `Record<string, SubListRow[]>` | widgetId → SubList 행 데이터 |
| `onSubListRowsChange` | `(widgetId: string, rows: SubListRow[]) => void` | SubList 행 변경 콜백 |

> PageGridRenderer 내부에서 자동으로 `GridCell → WidgetRenderer` 를 조립한다.  
> 직접 GridCell을 배치하는 코드를 작성할 필요가 없다.

---

### 4-4. WidgetRenderer

`widget.type`으로 **올바른 Renderer를 선택**한다. 직접 수정보다 확장이 원칙.

```tsx
// make/_shared/components/renderer/WidgetRenderer.tsx

// preview 모드
<WidgetRenderer
    mode="preview"          // 'preview' | 'live'
    widget={anyWidget}      // AnyWidget (type 필드로 분기)
    contentColSpan={12}     // RendererContainer 내부 grid 컬럼 수
/>

// live 모드 — 주요 props
<WidgetRenderer
    mode="live"
    widget={anyWidget}
    contentColSpan={12}
    /* Form 관련 */
    formValues={formValuesMap[wid] ?? {}}
    onFormValuesChange={(fieldId, value) => updateFormValue(wid, fieldId, value)}
    onContentAction={handleContentAction}   // (widgetIds: string[], action) => void
    /* SubList 관련 */
    subListRowsMap={subListRowsMap}         // Record<string, SubListRow[]>
    onSubListRowsChange={(wid, rows) => setSubListRowsMap(prev => ({ ...prev, [wid]: rows }))}
    /* 팝업 컨텍스트 */
    dataSlug="my-list"
    onRefresh={() => fetchData(0)}
/>
```

| widget.type | 연결 Renderer |
|---|---|
| `search` | SearchRenderer |
| `table` | TableRenderer |
| `form` | FormRenderer |
| `space` | SpaceRenderer |
| `category` | CategoryRenderer |
| `sublist` | SubListRenderer |

**live 모드 핵심 props 요약:**

| prop | 타입 | 용도 |
|---|---|---|
| `onContentAction` | `(connectedContentWidgetIds: string[], action: 'save' \| 'delete') => void` | Space 버튼이 Form·SubList widgetId 배열을 전달하며 저장/삭제 요청 |
| `subListRowsMap` | `Record<string, SubListRow[]>` | widgetId → SubList 행 데이터 (controlled) |
| `onSubListRowsChange` | `(widgetId: string, rows: SubListRow[]) => void` | SubList 행 변경 시 호출 |

**팝업 내부 SubList 처리 (WidgetRenderer 내부):**

```typescript
// WidgetRenderer 내부 state — 팝업이 열릴 때 초기화, 닫힐 때 소멸
const [popupSubListRowsMap, setPopupSubListRowsMap] = useState<Record<string, SubListRow[]>>({});

// 팝업 닫기 시 초기화
const handlePopupClose = () => {
    setPopupOpen(false);
    setPopupSubListRowsMap({});
};

// 팝업 본문 PageGridRenderer에 주입
<PageGridRenderer
    subListRowsMap={popupSubListRowsMap}
    onSubListRowsChange={(wid, rows) =>
        setPopupSubListRowsMap(prev => ({ ...prev, [wid]: rows }))
    }
/>
```

> `handlePopupContentAction`은 팝업 내 Form 위젯 뿐 아니라 SubList 위젯도 처리한다.

---

### 4-5. RendererContainer ⭐ 핵심

**모든 Renderer가 공통으로 사용하는 최상위 컨테이너.**  
테두리·배경·내부 grid를 한 곳에서 처리한다.

```tsx
// make/_shared/components/renderer/RendererContainer.tsx

// 기본 (테두리·배경만, grid 없음)
<RendererContainer showBorder bgColor="none">
  {children}
</RendererContainer>

// grid 배치 활성화 (Form, Space 등 필드를 grid로 배치할 때)
<RendererContainer contentColSpan={12} showBorder bgColor={bgColor}>
  <div style={{ gridColumn: 'span 6', gridRow: 'span 1' }}>필드A</div>
  <div style={{ gridColumn: 'span 6', gridRow: 'span 1' }}>필드B</div>
</RendererContainer>
```

**contentColSpan 전달 시 활성화되는 내부 grid:**
```typescript
{
    display: 'grid',
    gridTemplateColumns: `repeat(${contentColSpan}, 1fr)`,
    gridAutoRows: `${ROW_HEIGHT - GAP_SIZE}px`,   // 72px
    rowGap: `${GAP_SIZE}px`,                       // 8px
    columnGap: '8px',
}
```

> RendererContainer는 `h-full w-full`이므로 부모 GridCell의 전체 높이를 채운다.  
> 내부 grid가 GridCell 높이를 초과하면 `overflow: clip`에 의해 잘린다.

---

### 4-6. SubListRenderer ⭐ 동적 행 입력

Form 위젯과 함께 사용하는 **다건 행 입력 테이블** 렌더러.  
모든 행이 항상 입력 필드로 표시된다 (normal/editing 상태 구분 없음).

**타입 정의:**

```typescript
// make/_shared/components/renderer/SubListRenderer.tsx

/** 각 행의 데이터 — _rowId는 내부 식별용 (저장 시 제외) */
export interface SubListRow {
    _rowId: string;
    [key: string]: unknown;   // 컬럼 key → 값
}
```

**Props:**

```typescript
interface SubListRendererProps {
    mode: RendererMode;
    widget: SubListWidget;
    rows?: SubListRow[];                      // live 모드: 외부 controlled 상태
    onChange?: (rows: SubListRow[]) => void;  // 행 변경 시 호출
}
```

**사용 예시:**

```tsx
// preview — rows/onChange 불필요 (disabled 샘플 1행 자동 표시)
<SubListRenderer mode="preview" widget={subListWidget} />

// live — 외부 상태로 제어
<SubListRenderer
    mode="live"
    widget={subListWidget}
    rows={subListRowsMap[widget.widgetId] ?? []}
    onChange={rows => onSubListRowsChange(widget.widgetId, rows)}
/>
```

**모드별 동작:**

| 모드 | 동작 |
|---|---|
| `preview` | disabled 샘플 행 1개 표시. 추가/삭제 버튼 비활성 |
| `live` | + 버튼 → 빈 행 즉시 추가. 각 셀 입력 → 행 값 업데이트. 🗑️ → confirm 후 삭제 |

> `rows` prop은 항상 외부(generated/page.tsx)에서 `subListRowsMap`으로 관리한다.  
> SubListRenderer 내부에서 `useState`로도 보관하지만, 외부 prop이 변경되면 동기화된다.

## 5. 세 진입점에서의 조립 패턴

### ⭐ 핵심 규칙

> **UI는 컴포넌트 안에만 존재한다. 진입점(page.tsx)은 조립만 한다.**  
> 각 진입점에서 직접 UI를 그리거나 스타일을 하드코딩하면 즉시 불일치가 발생한다.

---

### 5-1. 진입점 A — builder-contents-layout (빌더 템플릿 레퍼런스)

새 컴포넌트를 만들 때 **가장 먼저 여기서 UI를 검증**한다.  
실제 데이터 없이 샘플 데이터로 컴포넌트 외관을 확인하는 공간이다.

```
파일: bo/src/app/admin/templates/builder-contents-layout/page.tsx
경로: /admin/templates/builder-contents-layout
```

```tsx
// builder-contents-layout/page.tsx 패턴
<PageLayout mode="preview">
    {TAB_CONFIG[activeTab].map((item, idx) => (
        <GridCell key={idx} colSpan={item.colSpan} rowSpan={item.rowSpan}>
            <WidgetRenderer
                mode="preview"
                widget={item.widget}
                contentColSpan={item.colSpan}
            />
        </GridCell>
    ))}
</PageLayout>
```

```typescript
// TAB_CONFIG 샘플 데이터 형식 — types.ts 타입 정의와 반드시 일치해야 함
const TAB_CONFIG = {
    폼: [
        {
            colSpan: 12,
            rowSpan: 13,
            widget: {
                type: 'form',
                widgetId: 'sample-form',
                fields: [...],  // FormWidget 타입 그대로
            } as FormWidget,
        },
    ],
};
```

> ⚠️ 샘플 데이터가 `types.ts` 타입 정의와 다르면 타입 오류가 발생한다.  
> 새 필드를 타입에 추가했다면 샘플 데이터도 반드시 맞춰 업데이트한다.

---

### 5-2. 진입점 B — 빌더 미리보기 (make/*/page.tsx)

좌측 설정 패널에서 편집한 내용이 **실시간으로 반영**되는 미리보기 영역.

```
파일: make/quick-detail/page.tsx, make/quick-list/page.tsx, make/widget/page.tsx
```

```tsx
// 빌더 미리보기 패턴 (quick-detail 예시)
{/* 우측: 미리보기 */}
<div className="flex-1 ...">
    <PageGridRenderer
        mode="preview"
        widgetItems={[                  // PageWidgetItem[]
            {
                id: 'wi-all',
                colSpan: 12,
                rowSpan: formContent.rowSpan + spaceContent.rowSpan,
                contents: [             // PageContentItem[]
                    {
                        id: formContent.id,
                        colSpan: formContent.colSpan,
                        rowSpan: formContent.rowSpan,
                        widget: formContent.widget,
                    },
                    {
                        id: spaceContent.id,
                        colSpan: spaceContent.colSpan,
                        rowSpan: spaceContent.rowSpan,
                        widget: spaceContent.widget,
                    },
                ],
            },
        ]}
    />
</div>
```

> PageGridRenderer 하나를 호출하면 내부에서 GridCell → WidgetRenderer 조립이 자동으로 이루어진다.  
> 미리보기에 UI를 직접 그리지 않는다.  
> preview 모드에서는 `onContentAction`, `subListRowsMap` 등 live 전용 props를 전달하지 않는다.

---

### 5-3. 진입점 C — 운영 페이지 (generated/*/page.tsx)

실제 사용자가 보는 **live 모드** 페이지.  
DB에서 불러온 템플릿 JSON을 그대로 PageGridRenderer에 전달한다.

```
파일: bo/src/app/admin/generated/{pageSlug}/page.tsx
```

```tsx
// 운영 페이지 — state
const [formValuesMap,  setFormValuesMap]  = useState<Record<string, Record<string, string>>>({});
const [subListRowsMap, setSubListRowsMap] = useState<Record<string, SubListRow[]>>({});

// 운영 페이지 — handleContentAction
// Space 버튼이 connectedContentWidgetIds(Form+SubList widgetId 배열)와 action을 전달
const handleContentAction = useCallback(async (
    connectedContentWidgetIds: string[],
    action: 'save' | 'delete'
) => {
    for (const widgetId of connectedContentWidgetIds) {
        // widgetId가 Form 위젯이면 formValuesMap에서 값 추출 후 저장/삭제
        // widgetId가 SubList 위젯이면 subListRowsMap에서 rows 추출 후 저장/삭제
        // sessionStorage에 formId_${widgetId} / sublistId_${widgetId} 키로 현재 ID 보관
    }
}, [widgetItems, formValuesMap, subListRowsMap, router]);

// 운영 페이지 — SubList 초기 로딩
// 페이지 진입 시 sessionStorage의 sublistId_${widgetId} 키로 기존 rows 조회
useEffect(() => {
    sublistWidgets.forEach(async (w) => {
        const storedId = sessionStorage.getItem(`sublistId_${w.widgetId}`);
        if (!storedId || !w.connectedSlug) return;
        const res = await api.get(`/page-data/${w.connectedSlug}/${storedId}`);
        setSubListRowsMap(prev => ({ ...prev, [w.widgetId]: res.data.dataJson.rows ?? [] }));
    });
}, []);

// 운영 페이지 — JSX
<PageLayout title={pageTitle} mode="live">
    <PageGridRenderer
        mode="live"
        widgetItems={widgetItems}           // DB에서 불러온 PageWidgetItem[] 배열
        formValuesMap={formValuesMap}
        onFormValuesChange={updateFormValue}
        onContentAction={handleContentAction}
        subListRowsMap={subListRowsMap}
        onSubListRowsChange={(wId, rows) => setSubListRowsMap(prev => ({ ...prev, [wId]: rows }))}
        codeGroups={codeGroups}
        /* 테이블·검색·카테고리 등 나머지 props ... */
    />
</PageLayout>
```

> `mode="live"`는 격자선 없음 + 실제 데이터 바인딩 + 인터랙션 활성화.  
> 컴포넌트 스택은 빌더 미리보기와 완전히 동일하다.  
> Form/SubList 저장 ID는 `sessionStorage`에 보관하여 수정·삭제 시 재사용한다.

## 6. UI 일관성 검증 방법

세 진입점이 동일하게 보이는지 확인하는 방법.

### 6-1. 브라우저 DevTools — grid-auto-rows 확인

```javascript
// 콘솔에서 실행 — 모든 grid-auto-rows 요소 측정
document.querySelectorAll('[style*="grid-auto-rows"]').forEach((rc, i) => {
    const style = window.getComputedStyle(rc);
    const rect  = rc.getBoundingClientRect();
    const children = Array.from(rc.children);
    const lastRect = children.at(-1)?.getBoundingClientRect();
    const clipping = lastRect ? Math.round(lastRect.bottom - rect.bottom) : 0;
    console.log(`RC[${i}] gridAutoRows=${style.gridAutoRows}, rowGap=${style.rowGap}`);
    console.log(`  → ${clipping > 0 ? `⚠️ ${clipping}px 클리핑` : `✅ 여유 ${-clipping}px`}`);
});
```

**정상 상태:**
```
RC[0] gridAutoRows=72px, rowGap=8px → ✅  (PageLayout)
RC[1] gridAutoRows=72px, rowGap=8px → ✅  (PageGridRenderer)
RC[2] gridAutoRows=72px, rowGap=8px → ✅  (Form RendererContainer)
RC[3] gridAutoRows=72px, rowGap=8px → ✅  (Space RendererContainer)
```

**이상 상태 (클리핑 발생 시):**
```
RC[2] gridAutoRows=80px, rowGap=0px → ⚠️ 8px 클리핑  ← RendererContainer 수정 필요
```

## 9. 검증 체크리스트

새 Renderer 또는 레이아웃 관련 변경 시 아래를 전부 확인한다.

- [ ] builder-contents-layout에 해당 위젯 탭이 있고, UI가 올바르게 표시되는가?
- [ ] 빌더 미리보기와 builder-contents-layout UI가 동일한가?
- [ ] 운영 페이지(live mode)와 빌더 미리보기 UI가 동일한가?
- [ ] 모든 Renderer가 `RendererContainer`를 사용하는가?
- [ ] `ROW_HEIGHT`, `GAP_SIZE` 상수를 import해서 사용하는가? (숫자 하드코딩 없는가?)
- [ ] RendererContainer의 `gridAutoRows`가 `ROW_HEIGHT - GAP_SIZE` (72px)인가?
- [ ] RendererContainer의 `rowGap`이 `GAP_SIZE` (8px)인가?
- [ ] GridCell의 `overflow: clip`으로 인한 컨텐츠 잘림이 없는가? (DevTools 확인)
- [ ] 새 Renderer를 `WidgetRenderer.tsx` switch 문에 등록했는가?
- [ ] PageGridRenderer에 `widgetItems`(PageWidgetItem[])를 전달하고 있는가? (`contents` 금지)
- [ ] live 모드에서 `onContentAction` 시그니처가 `(widgetIds: string[], action) => void` 인가?
- [ ] SubList 위젯이 있다면 `subListRowsMap` / `onSubListRowsChange`를 전달하고 있는가?
- [ ] 운영 페이지에서 SubList ID를 `sessionStorage`(`sublistId_${widgetId}`)에 보관하는가?
