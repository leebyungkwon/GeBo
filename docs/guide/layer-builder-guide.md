## 1. 화면 구성 개요

Layer Builder는 **등록 / 수정 / 상세 용도의 레이어 팝업**을 코드 없이 구성하는 도구입니다.
List Builder의 버튼이나 actions 컬럼에 연결되어 팝업으로 열립니다.

```
┌─────────────────────────────────────────────────────────────────┐
│  Layer Builder (/admin/templates/make/layer)                    │
│                                                                  │
│  ┌──────────────────┐  ┌──────────────────────────────────────┐ │
│  │   좌측 설정 패널  │  │          우측 미리보기               │ │
│  │                  │  │                                      │ │
│  │  팝업 유형 설정   │  │  실제 팝업 형태로 실시간 미리보기     │ │
│  │  필드 행 추가     │  │  (center: 중앙 모달 / right: 드로어)  │ │
│  │  하단 버튼 설정   │  │                                      │ │
│  └──────────────────┘  └──────────────────────────────────────┘ │
│                                                                  │
│  [저장/수정]  [생성]  버튼 → 우측 상단에 위치                    │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 팝업 기본 설정

### 팝업 유형

팝업의 표시 형태를 선택합니다.

| 유형 | 설명 | 적합한 용도 |
|---|---|---|
| **중앙 팝업 (center)** | 화면 정중앙에 오버레이 모달 형태 | 등록 / 수정 / 간단한 상세 |
| **우측 드로어 (right)** | 오른쪽에서 슬라이드인 패널 형태 | 상세 정보 / 넓은 폼이 필요한 경우 |

---

### 팝업 너비 (중앙 팝업)

중앙 팝업(`center`)에서만 적용됩니다. 우측 드로어는 고정 너비입니다.

| 옵션 | 너비 | 적합한 용도 |
|---|---|---|
| **Small (sm)** | 480px | 단순 확인 팝업, 단일 필드 입력 |
| **Medium (md)** | 672px | 일반적인 등록/수정 폼 (기본값) |
| **Large (lg)** | 768px | 필드가 많은 폼 |
| **X-Large (xl)** | 896px | 에디터 포함 등 넓은 공간이 필요한 폼 |

---

### 팝업 제목

팝업 상단에 표시되는 제목을 설정합니다.

- 등록용: `게시글 등록`
- 수정용: `게시글 수정`
- 상세용: `게시글 상세`

> 팝업을 등록/수정/상세 각각 별도 slug로 저장하는 경우, 제목도 용도에 맞게 구분하여 입력합니다.

## 3. 좌측 패널 — 폼 필드 구성

폼 필드를 **Row(행) 단위**로 구성합니다.
각 Row에는 1~5칸 컬럼 그리드가 있으며, 필드를 원하는 칸 수만큼 배치합니다.

### Row 추가 방법

1. **[+ 행 추가]** 버튼 클릭
2. 행 컬럼 수 선택 (1~5칸)
3. 행 안에 필드 추가 버튼 클릭 → 필드 타입 선택

---

### 폼 필드 타입별 상세

#### Input — 텍스트 입력

| 설정 항목 | 설명 |
|---|---|
| 라벨 | 필드 상단에 표시할 이름 |
| Key (fieldKey) | `data_json`에 저장될 키. **반드시 영문으로 입력** |
| Placeholder | 입력창 안에 표시할 안내 문구 |
| ColSpan | 차지할 그리드 칸 수 (1~5) |
| RowSpan | 세로 병합 칸 수 (1~3, 기본 1) |
| 필수 여부 | 저장 전 빈 값 검증 |
| 읽기 전용 | 상세 보기용. 입력 불가 상태로 표시 |
| 최소/최대 글자 수 | 입력 길이 검증 |
| 정규식 패턴 | 형식 검증 (예: `^[0-9]+$`) |
| 패턴 설명 | 오류 메시지 (예: "숫자만 입력") |

#### File — 파일 업로드

`page_file` 테이블에 파일을 업로드하고 연결합니다.

| 설정 항목 | 기본값 | 설명 |
|---|---|---|
| Key (fieldKey) | - | List의 file 컬럼 Key와 반드시 일치 |
| 최대 파일 수 | 1 | 동시에 업로드할 수 있는 파일 수 (1~20) |
| 개당 최대 용량 | 10 MB | 파일 1개의 최대 크기 (1~100 MB) |
| 전체 최대 용량 | 20 MB | 선택된 파일 전체 합계 최대 크기 |
| 허용 파일 유형 | 전체 허용 | 문서 / 이미지 / 동영상 / 커스텀 중 선택 |

**허용 파일 유형 프리셋:**

| 프리셋 | 허용 확장자 |
|---|---|
| **전체 허용** | 제한 없음 |
| **문서 (doc)** | `.pdf`, `.doc`, `.docx`, `.xls`, `.xlsx`, `.ppt`, `.pptx`, `.txt`, `.hwp` |
| **이미지 (image)** | `.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`, `.svg`, `.bmp` |
| **동영상 (video)** | `.mp4`, `.mov`, `.avi`, `.mkv`, `.webm`, `.wmv`, `.flv`, `.m4v` |
| **커스텀** | 직접 확장자 입력 (`.` 자동 추가, 중복 방지) |

---

#### Image — 이미지 업로드

이미지 파일을 업로드하고 미리보기로 표시합니다.

| 설정 항목 | 설명 |
|---|---|
| Key (fieldKey) | 저장 키. List의 file 컬럼 Key와 일치 필요 |
| ColSpan | 기본 2칸 (미리보기 공간 확보 권장) |
| 필수 여부 | 이미지 미첨부 시 오류 |

---

#### Video — 동영상

동영상을 URL 방식 또는 파일 업로드 방식으로 입력합니다.

| 설정 항목 | 설명 |
|---|---|
| Key (fieldKey) | 저장 키 |
| ColSpan | 기본 2칸 |
| RowSpan | 기본 2칸 (플레이어 공간 확보 권장) |
| 입력 방식 | URL 방식 또는 파일 업로드 방식 선택 |

**입력 방식:**

| 방식 | 설명 |
|---|---|
| **URL (url)** | YouTube / YouTube Shorts / Vimeo URL 입력. 자동으로 embed URL로 변환하여 미리보기 |
| **파일 업로드 (file)** | 동영상 파일 직접 업로드. `page_file` 테이블에 저장 |

지원 URL 형식 (자동 변환):
```
https://www.youtube.com/watch?v=ABC123  →  YouTube embed
https://youtu.be/ABC123                 →  YouTube embed
https://youtube.com/shorts/ABC123       →  YouTube Shorts embed
https://vimeo.com/123456                →  Vimeo embed
```

### 공통 설정 항목

모든 필드 타입에 공통으로 적용되는 항목입니다.

| 항목 | 설명 |
|---|---|
| **라벨** | 필드 상단에 표시되는 이름 |
| **Key (fieldKey)** | `data_json`에 저장되는 JSON 키. 미입력 시 라벨명 자동 사용 |
| **ColSpan** | 해당 필드가 차지할 그리드 가로 칸 수 (1~5) |

| **읽기 전용** | 상세 보기 팝업에서 값을 수정하지 못하게 잠금 |

> ⚠️ **Key는 항상 영문으로 입력하세요.**
> 미입력 시 한글 라벨명이 자동으로 Key가 되어, List 컬럼 Key와 불일치가 발생할 수 있습니다.

### 공통코드 연동

Select / Radio / Checkbox 타입은 옵션을 **공통코드 그룹**에서 자동으로 가져올 수 있습니다.

```
설정 방법:
1. 필드 편집 패널에서 [공통코드 연동] 드롭다운 선택
2. 원하는 코드 그룹 선택 (예: COUNTRY)
3. 해당 그룹의 활성화된 코드 목록이 옵션으로 자동 생성됨
```

| 방식 | 특징 |
|---|---|
| **수동 입력** | `표시이름:값` 형식으로 직접 입력. 코드 변경 시 빌더에서 다시 수정 필요 |
| **공통코드 연동** | 공통코드 관리에서 항목을 추가/수정하면 팝업에 자동 반영 |

## 4. 하단 버튼 구성

팝업 하단에 표시되는 버튼을 구성합니다.
기본으로 **닫기(secondary)** + **저장(primary)** 버튼이 제공됩니다.

### 버튼 타입

| 타입 | 스타일 | 설명 |
|---|---|---|
| `primary` | 검정 배경 흰 글씨 | 주요 액션 (기본 저장 버튼) |
| `secondary` | 흰 배경 테두리 | 보조/취소 버튼 (기본 닫기 버튼) |
| `blue` | 파란 배경 흰 글씨 | 정보성 액션 |
| `success` | 초록 배경 흰 글씨 | 확인/완료 버튼 |
| `danger` | 빨간 배경 흰 글씨 | 삭제/경고 버튼 |

### 버튼 액션

| 액션 | 동작 |
|---|---|
| **닫기 (close)** | 팝업을 닫습니다 |
| **저장 (save)** | validation 후 `POST` (신규) 또는 `PUT` (수정) API를 호출합니다 |
| **커스텀 (custom)** | 개발자방식 생성 파일에서 직접 로직을 구현할 때 사용 |

### 버튼 구성 예시

| 용도 | 버튼 구성 |
|---|---|
| 등록 팝업 | 닫기(secondary, close) + 등록(primary, save) |
| 수정 팝업 | 닫기(secondary, close) + 수정(primary, save) |
| 상세 팝업 (조회만) | 닫기(secondary, close) |
| 삭제 확인 팝업 | 취소(secondary, close) + 삭제(danger, save) |

## 5. Key 규칙 및 List 연결

Layer 필드의 **Key**는 `page_data.data_json`에 저장되는 JSON 키입니다.
List Builder의 테이블 컬럼 Key와 **정확히 일치**해야 데이터가 올바르게 표시됩니다.

### Key 흐름

```
[Layer Builder]          [page_data.data_json]       [List Builder]
필드 Key 입력         →  JSON 키로 저장           ←  컬럼 Key 입력
─────────────────────────────────────────────────────────────────────
"type"               →  "type":   "공지"          ←  Key: "type"
"title"              →  "title":  "첫 번째 글"    ←  Key: "title"
"regDate"            →  "regDate": "2026-04-13"   ←  Key: "regDate"
"attach" (file)      →  "attach": [1, 2]          ←  Key: "attach"
```

### Key 불일치 시 증상

| 증상 | 원인 |
|---|---|
| List 컬럼이 전부 비어 있음 | Layer Key와 List Key 불일치 |
| 파일이 조회되지 않음 | file 필드 Key와 List file 컬럼 Key 불일치 |
| 수정 팝업에 기존 값이 안 채워짐 | Layer Key와 data_json 실제 키 불일치 |

### file 필드 Key 주의

파일 업로드 필드의 Key는 `page_file.field_key`에도 저장됩니다.
List Builder의 `file` 타입 컬럼 Key와 반드시 동일해야 파일 개수가 올바르게 표시됩니다.

```
Layer file 필드 Key: attach
page_file.field_key:  attach  ← 자동으로 이 값으로 저장됨
List  file 컬럼 Key:  attach  ← 반드시 동일하게 입력
```

## 6. 저장/수정 — 관리자방식

빌더에서 구성한 팝업 설정을 **DB에 저장**하고,
런타임에 `LayerPopupRenderer` 컴포넌트가 DB에서 configJson을 읽어 팝업을 렌더링하는 방식입니다.

### DB 저장 흐름

```
1. 좌측 패널에서 팝업 유형 / 너비 / 제목 / 필드 / 버튼 구성 완료
2. 우측 상단 [저장/수정] 버튼 클릭
3. 저장 모달에서 이름 / slug / 설명 입력
4. [저장] 클릭 → DB page_template 테이블에 저장

저장 내용:
  - 이름: 게시판 등록 팝업
  - slug: board1
  - templateType: LAYER
  - configJson: { 팝업 유형/너비/제목/필드/버튼 전체 설정 JSON }
```

**slug 규칙:**
- 영문 소문자, 숫자, 하이픈(`-`)만 허용
- List Builder와 **연결 시 동일한 slug 사용**
- 등록/수정 팝업: `board1`
- 상세 팝업(read-only 별도 구성 시): `board1-detail`

---

### List Builder와 연결

저장된 LAYER 템플릿을 List Builder의 actions 컬럼 또는 버튼에 연결합니다.

**actions 컬럼 연결 (수정/상세):**

```
List Builder → 테이블 탭 → actions 컬럼 추가
  → 수정 버튼 팝업 slug: board1
  → 상세 버튼 팝업 slug: board1-detail
```

**버튼 연결 (등록):**

```
List Builder → 버튼 탭 → [+ 버튼 추가]
  → 버튼 이름: 등록
  → 액션: 레이어 팝업 열기
  → 팝업 slug: board1
```

**연결 후 동작:**

| 트리거 | 동작 |
|---|---|
| [등록] 버튼 클릭 | `LayerPopupRenderer(slug="board1")` → 빈 폼 표시 → 저장 시 `POST /api/v1/page-data/board1` |
| 행의 수정 아이콘 클릭 | `LayerPopupRenderer(slug="board1", editId=행ID, initialData=행데이터)` → 기존 값 채워진 폼 → 저장 시 `PUT /api/v1/page-data/board1/{id}` |
| 행의 상세 아이콘 클릭 | `LayerPopupRenderer(slug="board1-detail", editId=행ID)` → readonly 모드 (저장 버튼 없음) |

## 7. 생성 — 개발자방식

빌더에서 구성한 팝업 설정을 기반으로 **TSX 파일을 직접 생성**하는 방식입니다.
생성된 파일을 개발자가 수정하여 비즈니스 로직을 자유롭게 추가할 수 있습니다.

### TSX 파일 생성 흐름

```
1. 팝업 구성 완료
2. 우측 상단 [생성] 버튼 클릭
3. 생성 모달에서 slug 입력 (예: board1)
4. [생성] 클릭

생성되는 파일:
  bo/src/app/admin/generated/board1/LayerPopup.tsx
```

생성된 `LayerPopup.tsx`는:
- List Builder에서 같은 slug로 생성한 `page.tsx`가 직접 import하여 사용

```tsx
// page.tsx 내부 (List Builder에서 생성)
import LayerPopup from './LayerPopup';
```

> DB에는 저장하지 않습니다. 두 파일(`page.tsx` + `LayerPopup.tsx`)이 함께 동작합니다.

## 8. 기타 기능

### 템플릿 불러오기

설정 패널 상단 드롭다운에서 기존에 저장된 LAYER 템플릿을 불러올 수 있습니다.

```
- 드롭다운 클릭 → 저장된 LAYER 템플릿 목록 표시
- 검색창에 이름 입력 → 실시간 필터링
- 선택 → 해당 템플릿의 configJson이 빌더에 복원됨
```

불러온 후 수정하면 기존 템플릿 위에 **덮어쓰기(수정)** 또는 새 slug로 **신규 저장** 가능합니다.

---

### 생성 이력

[생성] 버튼으로 TSX 파일을 생성할 때마다 그 시점의 설정이 이력으로 저장됩니다.

```
- 상단 [생성 이력...] 드롭다운 클릭 → 과거 생성 이력 목록 표시
- 항목 선택 → 그 시점의 configJson이 빌더에 복원됨
- 항목 우측 [X] 클릭 → 해당 이력 삭제
```

---

### DB 연동 자동 구성

실제 DB 테이블을 선택하면 컬럼 목록을 읽어 필드를 자동으로 생성합니다.

```
설정 방법:
1. 설정 패널 상단 [DB 연동] 섹션에서 테이블 선택
2. 테이블의 컬럼 목록이 자동으로 폼 필드로 생성됨
3. 시스템 컬럼(reg_dt, mdf_dt 등 감사 컬럼)은 자동으로 제외됨
4. 생성된 필드를 필요에 맞게 수정 (타입 변경, 옵션 추가 등)
```

자동 제외되는 시스템 컬럼:

```
reg_dt, mdf_dt, reg_user_id, mdf_user_id
created_at, updated_at, created_by, updated_by
ins_dt, upd_dt, ins_user, upd_user
insert_dt, update_dt, insert_user, update_user
```
