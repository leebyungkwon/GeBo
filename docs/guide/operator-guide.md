## 1. 운영자방식이란?

빌더 화면에서 마우스 클릭만으로 목록 페이지와 팝업을 구성하고,
저장 버튼 하나로 **즉시 운영 화면에 반영**되는 방식입니다.
개발자에게 요청하거나 코드를 배포할 필요가 없습니다.

```
운영자가 할 일
  1. Layer 빌더에서 팝업 폼 구성 → [저장/수정]
  2. List 빌더에서 목록 화면 구성 → [저장/수정]
  3. 메뉴 관리에서 메뉴에 연결
  → 끝. 즉시 사이드바에 메뉴가 생기고 화면이 동작
```

**할 수 있는 것:**

| 기능 | 가능 여부 |
|---|---|
| 검색폼 구성 (Input, Select, 날짜범위 등) | ✅ |
| 테이블 컬럼 구성 (텍스트, 뱃지, 파일 등) | ✅ |
| 등록/수정/상세 팝업 구성 | ✅ |
| 메뉴 연결 | ✅ |
| 데이터 등록 / 수정 / 삭제 | ✅ |
| 엑셀 다운로드 | ✅ |
| 공통코드 연동 | ✅ |
| 코드 수정 없이 화면 변경 | ✅ |

## 2. STEP 1 — Layer 빌더로 팝업 구성

> 경로: `/admin/templates/make/layer`

목록 화면에서 **등록 / 수정 / 상세**에 사용할 팝업을 먼저 만듭니다.

---

### 1. 팝업 기본 설정

화면 좌측 상단에서 팝업 형태를 선택합니다.

| 설정 | 선택지 | 설명 |
|---|---|---|
| 팝업 유형 | 중앙 팝업 | 화면 정중앙에 모달 형태로 표시 |
| | 우측 드로어 | 오른쪽에서 슬라이드로 열리는 패널 |
| 팝업 너비 | Small ~ X-Large | 중앙 팝업에서만 적용 (내용이 많으면 Large 이상 권장) |
| 팝업 제목 | 직접 입력 | 팝업 상단에 표시될 제목 (예: 게시글 등록) |

---

### 2. 폼 필드 추가

**[+ 행 추가]** 버튼으로 행을 추가하고, 행 안에 필드를 배치합니다.

```
[+ 행 추가] 클릭
  → 행 컬럼 수 선택 (1~5칸)
  → 행 안의 [+ 필드 추가] 클릭
  → 필드 타입 선택
  → 우측 설정 패널에서 항목 입력
```

**필드 타입 선택 기준:**

| 입력 형태 | 사용할 타입 |
|---|---|
| 한 줄 텍스트 입력 | Input |
| 드롭다운 선택 | Select |
| 여러 줄 텍스트 | Textarea |
| 리치 텍스트 (서식 포함) | Editor |
| 날짜 선택 | Date |
| 하나만 선택 | Radio |
| 여러 개 선택 | Checkbox |
| 파일 첨부 | File |
| 이미지 첨부 | Image |
| 동영상 (URL 또는 파일) | Video |

**각 필드에서 반드시 입력할 항목:**

| 항목 | 설명 | 예시 |
|---|---|---|
| **라벨** | 필드 위에 표시되는 이름 | 제목, 작성자, 등록일 |
| **Key** | 데이터 저장 키. **반드시 영문으로 입력** | title, author, regDate |
| **ColSpan** | 가로 몇 칸 차지할지 | 제목은 4칸, 타입은 1칸 |
| **필수 여부** | 저장 전 빈 값 검증 필요 시 체크 | |

> ⚠️ **Key는 반드시 영문으로 입력하세요.**
> 한글로 입력하면 저장은 되지만 목록 화면에서 데이터가 표시되지 않습니다.

---

### 3. 하단 버튼 확인

기본으로 **닫기 + 저장** 버튼이 제공됩니다.
상세 보기 전용 팝업은 저장 버튼을 삭제하고 닫기만 남겨두세요.

---

### 4. 저장

우측 상단 **[저장/수정]** 버튼 클릭 → 저장 모달에서 입력합니다.

| 입력 항목 | 설명 | 예시 |
|---|---|---|
| 템플릿 이름 | 빌더 목록에서 식별할 이름 | 게시판 등록 팝업 |
| Slug | URL 및 데이터 식별자. **영문·숫자·하이픈만** | board1 |
| 설명 | 선택 입력 | 게시판 등록/수정 팝업 |

저장 완료 후 List 빌더로 이동합니다.

---

## 3. STEP 2 — List 빌더로 목록 화면 구성

> 경로: `/admin/templates/make/list`

---

### 1. 검색 탭 — 검색 필드 구성

**[+ 행 추가]** → 행 컬럼 수 선택 → 필드 추가 순서로 검색 조건을 구성합니다.

**검색 필드 타입 선택 기준:**

| 검색 방식 | 사용할 타입 |
|---|---|
| 텍스트 입력 검색 | Input |
| 드롭다운으로 선택 후 검색 | Select |
| 특정 날짜 검색 | Date |
| 날짜 범위 검색 (시작~종료) | DateRange |
| 라디오로 단일 선택 | Radio |
| 체크박스로 복수 선택 | Checkbox |
| 버튼으로 선택 | Button |

**DateRange 사용 시 주의:**

```
시작 라벨: 등록일 시작
종료 라벨: 등록일 종료
Key: regDate          ← 반드시 영문으로 입력
```

---

### 2. 테이블 탭 — 컬럼 구성

**[+ 컬럼 추가]** 버튼으로 테이블에 표시할 컬럼을 추가합니다.

**컬럼 타입 선택 기준:**

| 표시 내용 | 사용할 타입 |
|---|---|
| 일반 텍스트 / 숫자 | Text |
| 상태값을 색상 배지로 표시 | Badge |
| 공개/비공개 등 ON/OFF 표시 | Boolean |
| 수정 / 상세 / 삭제 버튼 | Actions |
| 첨부파일 개수 + 팝업 | File |

**컬럼 설정 시 반드시 확인:**

| 항목 | 설명 |
|---|---|
| **Key** | Layer 팝업의 같은 필드 Key와 **반드시 동일**해야 합니다 |
| **헤더명** | 테이블 상단에 표시되는 컬럼 이름 |

**Layer 팝업 연결 (Actions 컬럼):**

```
Actions 컬럼 추가
  → 수정 버튼 사용 여부: ON
  → 수정 팝업 slug: board1   ← STEP 1에서 저장한 slug 입력
  → 상세 버튼 사용 여부: ON (필요한 경우)
  → 상세 팝업 slug: board1-detail
  → 삭제 버튼 사용 여부: ON
```

---

### 3. 버튼 탭 — 버튼 구성

**[+ 버튼 추가]** 버튼으로 등록 버튼 등을 추가합니다.

```
버튼 추가
  → 버튼 이름: 등록
  → 버튼 타입: primary (검정)
  → 액션: 레이어 팝업 열기
  → 팝업 slug: board1   ← STEP 1에서 저장한 slug 입력
  → 위치: 검색폼 위 또는 아래 선택
```

---

### 4. 저장

우측 상단 **[저장/수정]** 버튼 클릭 → 저장 모달에서 입력합니다.

| 입력 항목 | 설명 | 예시 |
|---|---|---|
| 템플릿 이름 | 빌더 목록에서 식별할 이름 | 게시판 목록 |
| Slug | **Layer와 동일한 slug 입력** | board1 |
| 설명 | 선택 입력 | 게시판 목록 화면 |

> ⚠️ **List와 Layer의 slug를 동일하게 입력해야** 팝업이 올바르게 연결됩니다.

## 4. STEP 3 — 메뉴 연결

> 경로: `/admin/menus`

빌더에서 저장한 화면을 사이드바 메뉴에 연결합니다.

```
1. /admin/menus 접속
2. 연결할 메뉴 항목 클릭 (또는 [+ 메뉴 추가])
3. URL 입력란 옆 [페이지 메이커 연동] 버튼 클릭
4. 저장된 LIST 템플릿 목록에서 선택 (예: 게시판 목록)
5. URL 입력란에 /admin/generated/board1 자동 입력됨
6. [저장] 클릭
```

연결 완료 후:
- 사이드바에 메뉴가 바로 표시됩니다
- 메뉴 클릭 시 빌더에서 구성한 목록 화면이 열립니다
- 등록 버튼 클릭 시 Layer 팝업이 열립니다

---

### 전체 흐름 요약

```
Layer 빌더
  폼 필드 구성 (라벨 + Key + 타입)
  → [저장/수정] → slug: board1, 타입: LAYER

List 빌더
  검색 / 테이블 / 버튼 구성
  → Actions 컬럼 → 팝업 slug: board1
  → 등록 버튼 → 팝업 slug: board1
  → [저장/수정] → slug: board1, 타입: LIST

메뉴 관리
  [페이지 메이커 연동] → 게시판 목록 선택
  → URL: /admin/generated/board1 자동 입력
  → [저장]

결과
  사이드바 메뉴 클릭 → 목록 화면 표시
  [등록] 클릭 → 팝업 열림 → 저장 → 목록 자동 갱신
```

## 5. 데이터 관리

화면이 연결된 이후의 데이터 등록·수정·삭제·검색 방법입니다.

### 데이터 등록

```
목록 화면 → [등록] 버튼 클릭
  → Layer 팝업 열림
  → 각 필드에 값 입력
  → [저장] 클릭
  → 목록에 즉시 반영
```

### 데이터 수정

```
목록 화면 → 해당 행의 ✏️ (연필) 아이콘 클릭
  → Layer 팝업 열림 (기존 값이 채워진 상태)
  → 내용 수정
  → [저장] 클릭
  → 목록에 즉시 반영
```

### 데이터 삭제

```
목록 화면 → 해당 행의 🗑️ (휴지통) 아이콘 클릭
  → 확인 창 표시
  → [확인] 클릭
  → 목록에서 즉시 제거 (첨부파일도 함께 삭제)
```

### 검색

```
검색폼에 조건 입력
  → [검색] 버튼 클릭
  → 조건에 맞는 데이터만 목록에 표시

[초기화] 버튼 클릭
  → 검색 조건 전체 초기화
  → 전체 목록 재조회
```

### 엑셀 다운로드

버튼 탭에서 엑셀 다운로드 버튼을 추가한 경우:

```
[엑셀 다운로드] 버튼 클릭
  → 현재 검색 조건에 해당하는 전체 데이터를 엑셀 파일로 다운로드
  → 페이지네이션과 무관하게 전체 데이터 포함
```

## 6. 주의사항

### Key 일치 규칙

Layer 팝업의 필드 Key와 List 테이블의 컬럼 Key는 **반드시 동일**해야 합니다.

```
✅ 올바른 예
  Layer 필드 Key: title
  List 컬럼 Key:  title   ← 동일

❌ 잘못된 예
  Layer 필드 Key: title
  List 컬럼 Key:  제목    ← 불일치 → 목록에 데이터가 표시되지 않음
```

파일 컬럼도 동일합니다.

```
✅ 올바른 예
  Layer file 필드 Key: attach
  List file 컬럼 Key:  attach   ← 동일
```

---

### Key 영문 입력 규칙

Key는 **반드시 영문(소문자 권장), 숫자, 언더스코어(_)** 만 사용합니다.

```
✅ 올바른 예: title, regDate, user_name, attach1
❌ 잘못된 예: 제목, 등록일, 파일첨부
```

한글 Key를 입력하면:
- 데이터 저장은 되지만 목록에서 값이 표시되지 않습니다
- 날짜 범위 검색(DateRange)이 동작하지 않습니다

---

### Slug 규칙

Slug는 **영문 소문자, 숫자, 하이픈(-)** 만 사용합니다.

```
✅ 올바른 예: board1, notice-list, faq
❌ 잘못된 예: 게시판1, board 1 (공백 포함), Board1 (대문자)
```

Layer와 List의 slug를 동일하게 맞춰야 팝업이 올바르게 연결됩니다.

---

### 화면 수정 시

이미 운영 중인 화면의 빌더 설정을 변경하려면:

```
1. 해당 빌더 접속 (List 또는 Layer)
2. 상단 드롭다운에서 수정할 템플릿 선택 (불러오기)
3. 내용 수정
4. [저장/수정] 클릭 → 수정 모드로 저장
→ 저장 즉시 운영 화면에 반영
```

> ⚠️ 수정 후 저장하면 즉시 반영되므로, 운영 중인 화면은 신중하게 수정하세요.

---

### 템플릿 불러오기로 재활용

기존에 만든 화면 구성을 재활용하려면:

```
빌더 접속
  → 상단 드롭다운 클릭
  → 검색창에 이름 입력 또는 목록에서 선택
  → 불러오기 완료 → 원하는 부분 수정 후 새 slug로 저장
```
