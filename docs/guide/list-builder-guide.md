## 1. 화면 구성 개요

List Builder는 **검색폼 + 데이터 테이블** 형태의 목록 화면을 코드 없이 구성하는 도구입니다.

```
┌─────────────────────────────────────────────────────────────────┐
│  List Builder (/admin/templates/make/list)                      │
│                                                                  │
│  ┌──────────────────┐  ┌──────────────────────────────────────┐ │
│  │   좌측 설정 패널  │  │          우측 미리보기               │ │
│  │                  │  │                                      │ │
│  │  [검색] [테이블]  │  │  검색폼 + 테이블이 실시간으로 표시   │ │
│  │  [버튼]           │  │  샘플 데이터로 결과 확인 가능        │ │
│  │                  │  │                                      │ │
│  └──────────────────┘  └──────────────────────────────────────┘ │
│                                                                  │
│  [저장/수정]  [생성]  버튼 → 우측 상단에 위치                    │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 좌측 패널 — 검색 탭

검색폼을 **Row(행) 단위**로 구성합니다.
각 Row에는 1~5칸 컬럼 그리드가 있으며, 필드를 원하는 칸 수만큼 배치합니다.

### Row 추가 방법

1. **[+ 행 추가]** 버튼 클릭
2. 행 컬럼 수 선택 (1~5칸)
3. 행 안에 필드 추가 버튼 클릭 → 필드 타입 선택

---

### 검색 필드 타입별 상세

#### Input — 텍스트 입력

| 설정 항목 | 설명 |
|---|---|
| 라벨 | 필드 상단에 표시할 이름 |
| Key (fieldKey) | 검색 파라미터 키. **반드시 영문으로 입력** |
| Placeholder | 입력창 안에 표시할 안내 문구 |
| ColSpan | 차지할 그리드 칸 수 (1~5) |
| 필수 여부 | 체크 시 검색 전 빈 값 검증 |
| 최소 글자 수 | 이 글자 수 미만 입력 시 오류 |
| 최대 글자 수 | 이 글자 수 초과 입력 시 오류 |
| 정규식 패턴 | 입력값 형식 검증 (예: `^[0-9]+$`) |
| 패턴 설명 | 오류 메시지에 표시할 설명 (예: "숫자만 입력") |

---

#### Select — 드롭다운 선택

| 설정 항목 | 설명 |
|---|---|
| 라벨 | 필드 표시 이름 |
| Key (fieldKey) | 검색 파라미터 키. **반드시 영문으로 입력** |
| Placeholder | 기본 선택 문구 (예: "전체") |
| ColSpan | 그리드 칸 수 (1~5) |
| 필수 여부 | 미선택 시 검색 전 오류 |
| 옵션 입력 | `표시이름:값` 형식으로 줄바꿈 입력. 예: `공지:notice` |
| 공통코드 연동 | 코드 그룹 선택 시 옵션 자동 생성 (수동 입력과 택일) |

---

#### Date — 단일 날짜

| 설정 항목 | 설명 |
|---|---|
| 라벨 | 필드 표시 이름 |
| Key (fieldKey) | 검색 파라미터 키. **반드시 영문으로 입력** |
| ColSpan | 그리드 칸 수 (1~5) |
| 필수 여부 | 미입력 시 오류 |

---

#### DateRange — 날짜 범위

시작일 ~ 종료일을 한 필드로 입력받는 타입입니다.

| 설정 항목 | 설명 |
|---|---|
| 시작 라벨 | 시작 날짜 라벨 |
| 종료 라벨 | 종료 날짜 라벨 |
| Key (fieldKey) | 검색 파라미터 키. **반드시 영문으로 입력** ⚠️ |
| ColSpan | 기본 2칸 권장 (두 날짜 입력창이 나란히 표시됨) |
| 필수 여부 | 양쪽 모두 입력해야 통과 |

> ⚠️ **Key를 반드시 영문으로 입력해야 합니다.**
> Key 미입력 시 라벨(한글)이 그대로 파라미터 키로 사용되어 BE 검증에서 탈락 → 검색 결과 0건이 됩니다.

```
✅ 올바른 예: Key = regDate
❌ 잘못된 예: Key 미입력 → 라벨 "등록일 시작" 그대로 사용 → 검색 안 됨
```

**날짜 범위 검색 동작:**

| 입력 상태 | 검색 조건 |
|---|---|
| 시작일만 입력 | 시작일 이후 데이터 (`>=`) |
| 종료일만 입력 | 종료일 이전 데이터 (`<=`) |
| 둘 다 입력 | 시작일 이상 AND 종료일 이하 (`>= AND <=`) |
| 둘 다 미입력 | 날짜 조건 없음 (전체 조회) |

**Layer 폼 Key와 반드시 일치:**

검색 대상 날짜 데이터는 Layer 등록 폼에서 저장한 Key와 동일해야 합니다.

```
Layer 날짜 필드 Key:       regDate
List DateRange 필드 Key:  regDate  ← 반드시 동일하게 입력
```

#### Radio — 라디오 단일선택

| 설정 항목 | 설명 |
|---|---|
| 라벨 | 필드 표시 이름 |
| Key (fieldKey) | 검색 파라미터 키 |
| ColSpan | 그리드 칸 수 |
| 필수 여부 | 미선택 시 오류 |
| 옵션 입력 | `표시이름:값` 형식. 예: `사용:Y`, `미사용:N` |

항상 하나만 선택 가능합니다. 선택 시 이전 선택이 자동 해제됩니다.

---

#### Checkbox — 체크박스 복수선택

| 설정 항목 | 설명 |
|---|---|
| 라벨 | 필드 표시 이름 |
| Key (fieldKey) | 검색 파라미터 키 |
| ColSpan | 그리드 칸 수 |
| 필수 여부 | 미선택 시 오류 |
| 최소 선택 수 | 이 수 미만 선택 시 오류 |
| 최대 선택 수 | 이 수 초과 선택 시 오류 |
| 옵션 입력 | `표시이름:값` 형식 |
| 공통코드 연동 | 코드 그룹 선택 시 옵션 자동 생성 |

여러 항목 동시 선택 가능. 선택된 값들은 **쉼표 구분 문자열**로 저장됩니다.

```
한국 + 미국 + 중국 선택 → "KR,US,CN"
```

---

#### Button — 선택 버튼

| 설정 항목 | 설명 |
|---|---|
| 라벨 | 필드 표시 이름 |
| Key (fieldKey) | 검색 파라미터 키 |
| ColSpan | 그리드 칸 수 |
| 필수 여부 | 미선택 시 오류 |
| 다중선택 | ON: 여러 버튼 동시 선택 가능 / OFF(기본): 단일 선택 |
| 옵션 입력 | `표시이름:값` 형식. 예: `오늘:today`, `1주:1week` |
| 공통코드 연동 | 코드 그룹 선택 시 옵션 자동 생성 |

**다중선택 ON 시:**
- 여러 버튼을 동시에 선택할 수 있습니다
- 이미 선택된 버튼을 다시 클릭하면 선택이 해제됩니다
- 선택값은 쉼표 구분 문자열로 전달됩니다: `"KR,US"`

**다중선택 OFF 시 (기본):**
- 버튼 하나만 선택 가능
- 다른 버튼 클릭 시 이전 선택 자동 해제

**검색 API 전송 방식:**

다중선택 값은 쉼표 구분 문자열 그대로 검색 API에 전송됩니다.
BE에서 ILIKE 방식으로 처리되므로 선택한 값 중 하나라도 포함된 데이터가 조회됩니다.

```
한국 + 미국 선택 → API 파라미터: country=KR,US
→ data_json->>'country' ILIKE '%KR,US%' 조건으로 검색
```

### 공통 설정 항목

모든 필드 타입에 공통으로 적용되는 항목입니다.

| 항목 | 설명 |
|---|---|
| **라벨** | 필드 상단에 표시되는 이름 |
| **Key (fieldKey)** | BE로 전달되는 검색 파라미터 이름. 미입력 시 라벨명 자동 사용 |
| **ColSpan** | 해당 필드가 차지할 그리드 칸 수 (1~5) |
| **필수 여부** | 검색 전 빈 값을 검증. 빈 상태면 오류 메시지 표시 |

> ⚠️ **Key는 항상 영문으로 입력하세요.**
> 한글 라벨을 그대로 Key로 사용하면 BE `[a-zA-Z0-9_]+` 검증에서 탈락하여 검색이 동작하지 않습니다.

### 공통코드 연동

Select / Radio / Checkbox / Button 타입은 옵션을 **공통코드 그룹**에서 자동으로 가져올 수 있습니다.

```
설정 방법:
1. 필드 편집 패널에서 [공통코드 연동] 드롭다운 선택
2. 원하는 코드 그룹 선택 (예: COUNTRY)
3. 해당 그룹의 활성화된 코드 목록이 옵션으로 자동 생성됨
```

| 방식 | 특징 |
|---|---|
| **수동 입력** | `표시이름:값` 형식으로 직접 입력. 코드 변경 시 빌더에서 다시 수정 필요 |
| **공통코드 연동** | 공통코드 관리에서 항목을 추가/수정하면 자동 반영 |

### Validation 설정

검색 버튼 클릭 시 실행되는 입력값 검증입니다.

#### Input 전용

| 설정 | 설명 | 오류 메시지 예시 |
|---|---|---|
| 필수 여부 | 빈 값 금지 | `[필수] 이름` |
| 최소 글자 수 | 입력값이 이 수 이상이어야 함 | `[최소 2자] 이름 (현재 1자)` |
| 최대 글자 수 | 입력값이 이 수 이하여야 함 | `[최대 20자] 이름 (현재 25자)` |
| 정규식 패턴 | 형식 검증 (예: `^[0-9]+$`) | `[숫자만 입력] 전화번호` |

#### Checkbox 전용

| 설정 | 설명 | 오류 메시지 예시 |
|---|---|---|
| 필수 여부 | 최소 1개 선택 | `[필수] 카테고리` |
| 최소 선택 수 | N개 이상 선택해야 함 | `[최소 2개 선택] 카테고리 (현재 1개)` |
| 최대 선택 수 | N개 이하만 선택 가능 | `[최대 3개 선택] 카테고리 (현재 4개)` |

#### DateRange 전용

| 검증 조건 | 동작 |
|---|---|
| 필수 여부 ON | 시작일/종료일 둘 다 입력해야 함 |
| 종료일 < 시작일 | 자동 오류 (`[종료일이 시작일보다 빠름]`) |

## 3. 좌측 패널 — 테이블 탭

목록에 표시할 컬럼을 구성합니다. 각 컬럼은 드래그로 순서를 변경할 수 있습니다.

### 테이블 공통 설정

| 항목 | 설명 |
|---|---|
| 헤더명 | 컬럼 상단에 표시될 이름 |
| Key (accessor) | `data_json`에서 꺼낼 데이터 키. **Layer 필드 Key와 반드시 일치** |
| 너비 | 컬럼 너비 (px 또는 %) |
| 정렬 | 셀 내용 정렬 (left / center / right) |

---

### 테이블 컬럼 타입별 상세

#### text — 일반 텍스트

가장 기본적인 컬럼 타입. `data_json`에서 값을 꺼내 그대로 표시합니다.

| 설정 항목 | 설명 |
|---|---|
| Key (accessor) | 데이터 키 |
| 공통코드 연동 | 코드값을 이름으로 변환하여 표시 |
| 표시 방식 | `이름 표시` (기본) / `코드값 표시` |

**공통코드 연동 시 단일값 / 복수값 모두 지원:**

| 저장값 | 표시 결과 |
|---|---|
| `KR` (단일) | `한국` |
| `KR,US,CN` (복수) | `한국,미국,중국` |

복수값은 **checkbox** 또는 **button(다중선택)** 검색 필드로 저장된 데이터에서 발생합니다.

**설정 방법:**

테이블 컬럼 편집 패널에서:
1. **공통코드 연동** 드롭다운에서 해당 코드 그룹 선택
2. **표시 방식** → `이름 표시` 선택 (기본값)

`코드값 표시`를 선택하면 변환 없이 코드값 그대로 표시됩니다.

---

#### badge — 배지

값에 따라 색상이 다른 배지(태그)로 표시합니다.

| 설정 항목 | 설명 |
|---|---|
| Key (accessor) | 데이터 키 |
| 배지 옵션 | 각 값에 대한 표시이름 + 색상 매핑 |

배지 옵션 설정 예시:

| 값 | 표시이름 | 색상 |
|---|---|---|
| `notice` | 공지 | 파랑 |
| `event` | 이벤트 | 초록 |
| `general` | 일반 | 회색 |

매핑되지 않은 값은 기본 회색 배지로 표시됩니다.

---

#### boolean — 공개/비공개

`true` / `false` (또는 `Y` / `N`) 값을 시각적으로 표시합니다.

| 설정 항목 | 설명 |
|---|---|
| Key (accessor) | 데이터 키 |
| ON 표시 텍스트 | true 값일 때 표시 (기본: "공개") |
| OFF 표시 텍스트 | false 값일 때 표시 (기본: "비공개") |
| ON 색상 / OFF 색상 | 각 상태별 배지 색상 |

---

#### actions — 액션 버튼

행마다 수정 / 상세 / 삭제 버튼을 표시하는 컬럼입니다.
Key 입력이 불필요합니다.

| 설정 항목 | 설명 |
|---|---|
| 수정 버튼 | ON/OFF + 연결할 LAYER 팝업 slug 입력 |
| 상세 버튼 | ON/OFF + 연결할 LAYER 팝업 slug 입력 (보통 readonly 전용 LAYER 사용) |
| 삭제 버튼 | ON/OFF (클릭 시 confirm 후 DELETE API 호출) |
| 커스텀 버튼 | 커스텀 액션 버튼 추가 가능 (이름, 색상, LAYER slug 설정) |

**Layer 팝업 연결 방법:**

```
1. Layer Builder에서 팝업 구성 → slug 지정 (예: board1)
2. List Builder actions 컬럼 → 수정 버튼 slug: board1 입력
3. 목록에서 수정 아이콘 클릭 → board1 LAYER 팝업이 해당 행 데이터와 함께 열림
```

---

#### file — 첨부파일

`page_file` 테이블에 연결된 파일 개수를 📎 아이콘과 함께 표시합니다.
클릭 시 파일 목록 팝업이 열립니다.

| 설정 항목 | 설명 |
|---|---|
| Key (accessor) | **Layer 파일 필드 Key와 반드시 동일하게 입력** |

> ⚠️ Layer의 file 필드 Key와 List의 file 컬럼 Key가 다르면 파일 조회가 되지 않습니다.

```
Layer 파일 필드 Key: attach
List  file 컬럼 Key: attach  ← 반드시 동일
```

---

### Key 규칙 및 Layer 연결

테이블 컬럼의 Key는 Layer 필드의 Key와 **정확히 일치**해야 합니다.
이 Key가 `page_data.data_json`의 JSON 키로 사용되기 때문입니다.

```
[Layer Builder]          [page_data.data_json]       [List Builder]
필드 Key 입력         →  JSON 키로 저장           ←  컬럼 Key 입력
─────────────────────────────────────────────────────────────────────
"type"               →  "type":   "공지"          ←  Key: "type"
"title"              →  "title":  "첫 번째 글"    ←  Key: "title"
"attach" (file)      →  "attach": [1, 2]          ←  Key: "attach"
```

**Key 불일치 시 증상:**

| 증상 | 원인 |
|---|---|
| 컬럼이 비어 있음 (`-`) | List Key와 Layer Key 불일치 |
| 파일 개수가 0으로 표시 | file 컬럼 Key와 Layer 파일 Key 불일치 |

---

## 4. 좌측 패널 — 버튼 탭

목록 상단 또는 검색폼 하단에 표시될 버튼을 구성합니다.

### 버튼 타입

| 타입 | 스타일 | 설명 |
|---|---|---|
| `primary` | 검정 배경 흰 글씨 | 주요 액션 버튼 (예: 등록) |
| `secondary` | 흰 배경 테두리 | 보조 버튼 (예: 초기화) |
| `blue` | 파란 배경 흰 글씨 | 정보/조회 관련 버튼 |
| `success` | 초록 배경 흰 글씨 | 확인/완료 관련 버튼 |
| `danger` | 빨간 배경 흰 글씨 | 삭제/경고 버튼 |

---

### 버튼 액션 및 Layer 연결

| 액션 | 설명 |
|---|---|
| **레이어 팝업 열기** | 지정한 LAYER 팝업 slug를 열어 등록 폼을 표시 |
| **엑셀 다운로드** | 현재 목록 데이터를 엑셀 파일로 다운로드 |
| **커스텀** | 개발자가 코드에서 직접 로직을 구현할 때 사용 |

**레이어 팝업 연결 예시 (등록 버튼):**

```
1. Layer Builder에서 등록 폼 구성 → slug: board1 (LAYER)
2. List Builder 버튼 탭 → [+ 버튼 추가]
3. 버튼 이름: 등록, 타입: primary
4. 액션: 레이어 팝업 열기, 팝업 slug: board1
5. 저장 후 목록 페이지에서 [등록] 버튼 클릭 → board1 LAYER 팝업 오픈
```

---

### 버튼 위치

| 위치 | 설명 |
|---|---|
| **검색폼 위 (above-search)** | 검색폼보다 위에 표시. 테이블 바로 위에 배치됨 |
| **검색폼 아래 (below-search)** | 검색폼 아래, 테이블 위에 표시 |

## 5. 저장/수정 — 관리자방식

빌더에서 구성한 설정을 **DB에 저장**하고, 공통 렌더러(`[slug]/page.tsx`)가 런타임에 읽어 페이지를 표시하는 방식입니다.
코드 배포 없이 저장 즉시 반영됩니다.

### DB 저장 흐름

```
1. 좌측 패널에서 검색 / 테이블 / 버튼 구성 완료
2. 우측 상단 [저장/수정] 버튼 클릭
3. 저장 모달에서 이름 / slug / 설명 입력
4. [저장] 클릭 → DB page_template 테이블에 저장

저장 내용:
  - 이름: 게시판 목록1
  - slug: board1
  - templateType: LIST
  - configJson: { 검색필드/테이블/버튼 전체 설정 JSON }
```

**slug 규칙:**

- 영문 소문자, 숫자, 하이픈(`-`)만 허용
- 예: `board1`, `notice-list`, `user-management`
- 이 slug가 URL 경로 및 데이터 조회 키로 사용됨

---

### 메뉴 연결

DB에 저장된 LIST 템플릿을 메뉴에 연결하여 사이드바에 표시합니다.

```
1. /admin/menus 접속 (메뉴 관리)
2. 연결할 메뉴 항목 클릭 (또는 [+ 메뉴 추가])
3. URL 입력란 옆 [페이지 메이커 연동] 버튼 클릭
4. 저장된 LIST 템플릿 목록 드롭다운에서 선택 (예: 게시판 목록1)
5. URL 입력란에 /admin/generated/board1 자동 입력됨
6. [저장] 클릭
```

연결 후:
- 사이드바에 메뉴가 표시됨
- 메뉴 클릭 → `/admin/generated/board1`으로 이동
- 공통 렌더러(`[slug]/page.tsx`)가 DB에서 configJson을 읽어 페이지 자동 렌더링

---

## 6. 생성 — 개발자방식

빌더에서 구성한 설정을 기반으로 **TSX 파일을 직접 생성**하는 방식입니다.
생성된 파일을 개발자가 수정하여 비즈니스 로직을 자유롭게 추가할 수 있습니다.

### TSX 파일 생성 흐름

```
1. 좌측 패널에서 검색 / 테이블 / 버튼 구성 완료
2. 우측 상단 [생성] 버튼 클릭
3. 생성 모달에서 slug 입력 (예: board1)
4. [생성] 클릭

생성되는 파일:
  bo/src/app/admin/generated/board1/page.tsx
```

생성된 `page.tsx`는:
- slug 기반 `page_data` CRUD를 그대로 사용 (관리자방식과 데이터 구조 동일)
- 만약 같은 slug로 Layer도 생성했다면 `LayerPopup.tsx`를 직접 import

```tsx
// 생성된 page.tsx 내부
import LayerPopup from './LayerPopup';
```

---

### 메뉴 연결

개발자방식은 DB에 템플릿을 저장하지 않으므로 **[페이지 메이커 연동] 버튼을 사용할 수 없습니다.**
메뉴 관리 URL 입력란에 직접 입력합니다.

```
1. /admin/menus 접속
2. 연결할 메뉴 항목 클릭
3. URL 입력란에 직접 입력: /admin/generated/board1
4. [저장] 클릭
```

---

## 7. 기타 기능

### 템플릿 불러오기

설정 패널 상단 드롭다운에서 기존에 저장된 LIST 템플릿을 불러올 수 있습니다.

```
- 드롭다운 클릭 → 저장된 LIST 템플릿 목록 표시
- 검색창에 이름 입력 → 실시간 필터링
- 선택 → 해당 템플릿의 configJson이 빌더에 복원됨
```

불러온 후 수정하면 기존 템플릿 위에 **덮어쓰기(수정)** 또는 새 slug로 **신규 저장** 가능합니다.

---

### 생성 이력

[생성] 버튼으로 TSX 파일을 생성할 때마다 그 시점의 설정이 이력으로 저장됩니다.

```
- 상단 [생성 이력...] 드롭다운 클릭 → 과거 생성 이력 목록 표시
- 항목 선택 → 그 시점의 configJson이 빌더에 복원됨
- 항목 우측 [X] 클릭 → 해당 이력 삭제
```

과거 설정으로 롤백하거나 특정 시점의 구성을 재활용할 때 유용합니다.
